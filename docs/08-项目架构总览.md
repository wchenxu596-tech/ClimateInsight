# ClimateInsight 项目架构总览

> 目标读者：AI / 新开发者，仅凭本文即可理解整个项目的完整结构和运行方式。

---

## 1. 项目简介

**ClimateInsight** 是一个基于 NOAA GSOD 全球气象数据的 BI 分析平台。支持 2015-2025 共 11 年的全球气象数据查询、可视化大屏展示、AI 智能问答。

- **技术栈**：Python Flask（后端） + Vue 3 / Element Plus / ECharts（前端） + MySQL（数据库）
- **数据来源**：NOAA Global Summary of the Day (GSOD) https://www.ncei.noaa.gov/data/global-summary-of-the-day/archive/
- **数据链路**：NOAA tar.gz 归档 → Python ETL 流水线 (`download_and_load.py`，下载+预处理+聚合一站式) → MySQL DWS/ADS 层 → Flask API → Vue 3 前端

## 3. 数据库设计（MySQL: climate_dw）

### 3.1 DWS 层 — dws_station_monthly（气象站月度汇总）

| 字段           | 类型         | 说明                                              |
| -------------- | ------------ | ------------------------------------------------- |
| station_id     | VARCHAR(20)  | 气象站 ID                                         |
| station_name   | VARCHAR(200) | 气象站名称                                        |
| year           | INT          | 年份                                              |
| obs_month      | INT          | 月份 (1-12)                                       |
| latitude       | DOUBLE       | 纬度                                              |
| longitude      | DOUBLE       | 经度                                              |
| avg_temp       | DOUBLE       | 月均温 (°C)                                     |
| avg_temp_max   | DOUBLE       | 月均最高温                                        |
| avg_temp_min   | DOUBLE       | 月均最低温                                        |
| max_temp       | DOUBLE       | 月最高温                                          |
| min_temp       | DOUBLE       | 月最低温                                          |
| total_precip   | DOUBLE       | 月总降水 (mm)                                     |
| rainy_days     | INT          | 降雨天数                                          |
| extreme_days   | INT          | 极端天气天数                                      |
| heat_wave_days | INT          | 热浪天数 (>=35 degC)                              |
| cold_wave_days | INT          | 寒潮天数 (<=-10 degC)                             |
| frost_days     | INT          | 霜冻天数                                          |
| snow_days      | INT          | 降雪天数                                          |
| thunder_days   | INT          | 雷暴天数                                          |
| obs_days       | INT          | 观测天数                                          |
| climate_zone   | VARCHAR(20)  | 气候带 (tropical/temperate/continental/polar/arid) |

主键: `(station_id, year, obs_month)`
索引: `idx_year_month`, `idx_climate_zone`, `idx_temp_max`, `idx_precip`

### 3.2 ADS 层 — 6 张应用表

| 表名              | 主键                                                      | 用途                                      |
| ----------------- | --------------------------------------------------------- | ----------------------------------------- |
| ads_kpi           | (kpi_name, data_year)                                     | 全球 KPI：年均温、站点数、极端占比、最高温 |
| ads_monthly_trend | (data_year, obs_month)                                    | 12 个月全球均温 / 均最高 / 均最低          |
| ads_ranking       | AUTO_INCREMENT id + UNIQUE(category, data_year, rank_num) | 4 类排名各 TOP15                            |
| ads_zones         | (data_year, climate_zone)                                 | 5 个气候带站点数                           |
| ads_zone_trends   | (data_year, climate_zone)                                 | 气候带年度趋势（均温/降水/极端/热浪/寒潮） |
| ads_stations      | (station_id, data_year)                                   | 站点年度概要（15 列，加速地图/预警页面）    |

**ads_stations 表结构（15 列）**：`data_year, station_id, station_name, latitude, longitude, climate_zone, avg_temp, total_precip, risk_events, heat_wave, cold_wave, extreme, frost, snow, thunder`

## 4. 后端架构

### 4.1 入口与配置

- `app.py`：创建 Flask 应用，注册 6 个 Blueprint，配置 CORS，提供 SPA 静态文件 fallback
- `config.py`：所有配置从环境变量读取，通过 `python-dotenv` 加载 `.env` 文件。包含 `DATA_YEAR` 默认年份、`AGENT_ENABLED` 开关、LLM API 配置
- `db.py`：pymysql 连接封装。`query(sql, params)` 返回元组列表，`query_dict()` 返回字典列表。每次查询新建连接
- `cache.py`：内存缓存装饰器 `@cached(300)`，按 URL+函数名 MD5 缓存 JSON 响应，5 分钟 TTL。超 200 条自动清理

### 4.2 API 端点（共 13 个）

| 方法 | 路径                 | 参数                        | 说明                                       |
| ---- | -------------------- | --------------------------- | ------------------------------------------ |
| GET  | /api/health          | --                          | 健康检查，返回 DB 状态                     |
| GET  | /api/kpi             | year                        | 4 项 KPI 指标                              |
| GET  | /api/monthly         | year                        | 12 个月温度数据                            |
| GET  | /api/zones           | year                        | 5 个气候带站点数                           |
| GET  | /api/ranking         | year, category, limit       | 站点排名（4 类别）                         |
| GET  | /api/trend           | year                        | 多站点月度趋势（支持 station_id 筛选）     |
| GET  | /api/alert/risk      | year, limit                 | 极端天气风险评分（四级预警）               |
| GET  | /api/alert/stations  | year, zone                  | 气候带站点预警列表                         |
| GET  | /api/alert/zones     | year                        | 各气候带预警统计                           |
| GET  | /api/stations        | year, zone, region          | 地图站点数据（支持气候带/洲洋筛选）        |

所有成功响应格式：`{"code":0, "message":"ok", "data":[...], "meta":{"data_year":2025}}`

### 4.3 AI Agent 架构

```
用户问题 + UI 上下文 (year, page)
    │
    ▼
agent.py（路由层）
    ├─ 输入校验：非 JSON -> 400, 空问题 -> 400, >300 字 -> 400, SQL 关键词 -> 400
    ├─ 年份校验：非整数 -> 400, 不在 {2015..2025} -> 400
    │
    ▼
service.py（编排层）
    │
    ▼
intents.py（意图识别 — 双引擎）
    ├─ detect_llm() -> DeepSeek API（temperature=0, max_tokens=250）
    │   ├─ System Prompt 严格约束输出为 14 种 JSON 格式之一
    │   ├─ 解析 JSON -> Schema 校验（字段完整性 + 类型检查）
    │   ├─ 校验失败 -> 返回 None
    │   └─ 超时 5 秒 -> 返回 None
    │
    └─ detect_rules() -> 规则引擎（关键词 + 正则，兜底方案）
        └─ 年份检测、月份范围、排名类别等确定性提取
    │
    ▼
service.py（工具调度）
    ├─ 无工具意图 (chat/help/unknown) -> 直接模板回答
    ├─ 单工具意图 (kpi/monthly/ranking/zones) -> 调用对应 TOOL
    ├─ 比较意图 (compare) -> 多年代理调用
    ├─ 趋势分析 (trend_analysis) -> 多年趋势 + 历史 KPI
    ├─ 季节分析 (seasonal) -> 最近 4 年趋势
    ├─ 气候带趋势 (zone_detail) -> 多年气候带演变
    ├─ 极端天气 (extremes) -> 极端事件趋势
    ├─ 站点查询 (station_query) -> 模糊匹配 + 月度明细
    └─ 页面分析 (page_analysis) -> 富数据拉取（KPI + 历史 + 排名 + 气候带）
    │
    ▼
tools.py（工具白名单 — 8 个函数）
    ├─ get_kpi(year)              -> SELECT ads_kpi WHERE data_year=%s
    ├─ get_monthly(year)          -> SELECT ads_monthly_trend WHERE data_year=%s
    ├─ get_ranking(y, c, l)       -> SELECT ads_ranking WHERE data_year=%s AND category=%s LIMIT %s
    ├─ get_zones(year)            -> SELECT ads_zones WHERE data_year=%s
    ├─ get_kpi_history(years)     -> 多年 KPI 批量查询
    ├─ get_trend_multi_year(years)-> 多年温度趋势批量查询
    ├─ get_zones_trend(years)     -> 多年气候带趋势（ads_zone_trends）
    └─ get_station_detail(id, y)  -> 单站月度明细
    │
    ▼
responder.py（模板回答生成）
    ├─ kpi -> 文本 + 表格
    ├─ monthly -> 文本 + line 图（支持月份范围过滤）
    ├─ ranking -> 文本 + 表格 + bar 图（站名自动翻译中文）
    ├─ zones -> 文本 + 表格 + pie 图
    ├─ compare -> 多年对比文本 + 表格
    ├─ trend_analysis -> 趋势分析 + 多年折线图
    ├─ seasonal -> 季节变化模式分析 + 图
    ├─ zone_detail -> 气候带演变 + 图
    ├─ extremes -> 极端天气趋势 + 图
    ├─ station_query -> 站点详情 + 月度明细表
    ├─ page_analysis -> 页面上下文智能分析 + 多图表
    ├─ chat -> 自我介绍
    └─ help -> 网站使用指南
```

**安全设计**：

- LLM 只输出 JSON 意图，不生成 SQL / 代码 / 文本回答
- Planner 支持 LLM Function Calling 输出受控的 AnalysisPlan（catalog.py + schemas.py 双重校验）
- 工具层仅访问 ADS / DWS 表，全部参数化查询
- JSON Schema 校验拒绝非法字段
- 任何环节失败都回退到规则引擎

## 5. 前端架构

### 5.1 技术栈

- **Vue 3** (Composition API + `<script setup>`)
- **Element Plus** (UI 组件库：表格、输入框、标签、弹窗)
- **ECharts** (图表：折线图、柱状图、饼图、散点图、地图，按需加载渲染器)
- **Vue Router** (2 条路由，HTML5 History 模式：首页 scroll-snap + 详情页)
- **Axios** (HTTP 请求，baseURL 默认 `/api`，15 秒超时，前端 30s 缓存)
- **@bg-effects/grainient** (WebGL 动态渐变背景)

```
│            │                              │            │
│  左侧边栏   │        主内容区               │  右侧导航   │

│            │  │                      │    │  + AI 开关  │
│  2015      │  │  <router-view>       │    │            │

│            │  └────────────────────────┘ │            │

```

关键特性：
- **Grainient WebGL 背景**：绿-青蓝-暖橙三色动态渐变（`#bceecf, #bae8ef, #ffdbce`）
- **玻璃拟态 (Glassmorphism)**：所有卡片 `backdrop-filter: blur(12px)`，半透明背景 + 微边框
- **Scroll-snap 首页**：6 个 `<section>` 全屏吸附滚动（`scroll-snap-type: y mandatory`）
- **AI 面板**：右侧滑入面板，宽度可拖拽调整（默认 400px），`Transition` 动画
- **页面过渡**：`page-enter-from` / `page-leave-to` CSS 动画（fade + 上移）
- **年份全局化**：通过 Vue `provide/inject` 传递，App.vue 提供 `selectedYear`，所有页面 `watch` 变化自动重新加载

## 6. 数据导入流程

### 6.1 一站式流水线（download_and_load.py）

```bash
# 单年导入（下载 + 预处理 + 聚合 + 写入 MySQL，全自动）
export MYSQL_PASSWORD=123456
python scripts/download_and_load.py --year 2025

# 批量导入所有年份
python scripts/import_all_years.py
```

### 6.2 流水线详解

```
步骤 1: 下载
  curl NOAA GSOD 年度归档 (https://...archive/{year}.tar.gz)
  -> data/raw/{year}.tar.gz (~500MB)

步骤 2: 解压 + 预处理（流式，内存友好）
  tar.gz -> 逐个 CSV
  -> degF -> degC, 英寸 -> mm, 天气标志解析 (FRSHTT)
  -> 质量标记过滤

步骤 3: 聚合（Python 内存聚合）
  -> 按 (station_id, year, month) 分组
  -> 计算均温、最值、降水、热浪/寒潮/霜冻/降雪/雷暴天数
  -> 气候带分类（基于纬度 + 温度 + 降水规则）

步骤 4: DWS 写入
  -> DELETE + INSERT 当前年份
  -> 批量 INSERT (500 条/批) 到 dws_station_monthly

步骤 5: ADS 计算 + 写入
  -> ads_kpi: 全球年均温、站点总数、极端占比、最高单站温度
  -> ads_monthly_trend: 12 个月全球均温/均最高/均最低
  -> ads_ranking: 4 类别各 TOP15
  -> ads_zones: 5 个气候带站点数
  -> ads_zone_trends: 各气候带年度趋势（均温/降水/极端/热浪/寒潮）
  -> ads_stations: 站点年度概要（15 列预聚合）
```

### 6.3 数据校验

```bash
python scripts/verify_data.py   # 数据质量校验（行数、温度范围、站点覆盖等）
```

## 9. 关键设计决策

1. **一站式数据流水线**：`download_and_load.py` 将下载、预处理、DWS 聚合、ADS 计算整合为单一脚本，直接替代旧的多脚本流程（etl_csv_to_mysql.py + preprocess_climate.py 等），支持一键全自动导入
2. **LLM 作为分类器**：大模型只输出结构化 JSON 意图（或受控 AnalysisPlan），不生成 SQL 或自然语言回答。回答由 `responder.py` 模板引擎生成。所有 SQL 由 `tools.py` 白名单执行
3. **三层安全防护**：API 输入过滤 -> JSON Schema 校验 -> 工具白名单（仅参数化查询 ADS/DWS 表）
4. **侧边 AI 面板**：AI 助手作为右侧滑入面板，替代旧版全局悬浮弹窗。通过 Vue `provide/inject` 获取当前年份和页面上下文，支持宽度拖拽调整
5. **3 列 CSS Grid 主布局**：`80px 1fr 80px` 三列布局，左列年份选择、中间主内容、右列页面导航 + AI 开关。首页内嵌 6 个 scroll-snap 全屏页面
6. **Grainient 动态背景**：WebGL 驱动的绿-青蓝-暖橙三色渐变 + 细粒度噪点，替代静态背景
7. **玻璃拟态设计语言**：全局应用 `backdrop-filter: blur(12px)` + 半透明背景 + 微边框，卡片悬停上浮 + 光泽扫过
8. **ADS 复合主键**：`(kpi_name, data_year)`, `(data_year, obs_month)`, `(station_id, data_year)` 等支持多年数据共存，替代旧版 `AUTO_INCREMENT` 主键
9. **年份选择器左侧化**：年份从顶部选择器移至左侧固定侧边栏，垂直滚动可选 2015-2025 共 11 年，自动滚动到当前选中年份
10. **6 张 ADS 表**：在原有 4 张 ADS 表基础上新增 `ads_zone_trends`（气候带年度趋势聚合）和 `ads_stations`（15 列站点年度概要），加速地图渲染和预警查询
11. **Agent 模块化升级**：从 5 个模块扩展到 10 个（新增 catalog、planner、schemas、statistics、alerts_engine、chart_spec），支持统计计算、预警引擎、受控图表生成
