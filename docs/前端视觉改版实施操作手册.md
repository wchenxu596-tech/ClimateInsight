# ClimateInsight 前端视觉改版实施操作手册

## 1. 目的、范围与实施结论

本手册用于指导其他 AI 或开发者，将 `前端.txt` 中的三套静态设计稿落地到既有 ClimateInsight 前端，而不是把设计稿中的 HTML、Tailwind CDN 和模拟数据直接复制进项目。

本次改版的目标是统一总览、趋势、排名、气候带和 AI 助手的视觉语言：自然风景背景、低透明度毛玻璃面板、森林绿主色、青绿色辅助色、陶土橙风险色、较大的留白和清晰的数据层级。既有的 Vue 3 + Element Plus + ECharts + Axios 架构、路由、API 契约和数据加载逻辑必须保留。

### 1.1 已确认的设计稿覆盖范围

`前端.txt` 实际包含以下三个页面稿：

| 设计稿 | 对应现有页面 | 重点 |
|---|---|---|
| Dashboard | `Dashboard.vue` | KPI、月度趋势、气候带分布、TOP 15、圆形 AI 入口 |
| Rankings | `CityRanking.vue` | 排名分段选择、图表、明细表、AI 面板视觉 |
| Trend Analysis | `TrendAnalysis.vue` | 左侧筛选、主趋势图、三张指标卡、AI 面板视觉 |

没有提供 `ClimateZones.vue` 的单独稿。因此必须采用 Dashboard 中“Climate Zone Distribution”的环形图和同一套玻璃面板样式延展该页，不得凭空增加地图、告警中心或设置页。

### 1.2 必须遵守的决定

1. **不引入 Tailwind CDN。** 静态稿中的 Tailwind 仅是视觉参考；生产项目继续使用 Vue 的 scoped CSS 和公共 CSS 变量。
2. **不把设计稿中的示例数值写死。** 所有图表和 KPI 均使用现有 API 返回的数据。
3. **不伪造接口没有提供的数据。** 例如“1990–2020 基准”“国家”“最高记录温度”“35°C 以上天数”“98% 运行率”“告警”“导出历史”，没有真实数据或接口时必须隐藏、禁用或标记为“暂未接入”。
4. **不使用设计稿中的 Google 图片直链作为生产资源。** 这些链接不可控且授权不明。使用项目内已获授权的背景图；未提供图片时使用文中规定的 CSS 渐变备用背景。
5. **中文为产品主语言。** 沿用项目现有中文导航、标题、错误提示和 API 数据；仅保留 `ClimateInsight` 品牌名与 `NOAA GSOD` 数据源名。
6. **不改变现有 API 路径和返回格式。** 前端仍从 `/api` 读取 `kpi`、`monthly`、`zones`、`ranking`、`agent/query`。

## 2. 实施前检查

在开始改代码前，执行并确认以下项目。若与本手册的路径不同，以实际仓库路径为准，但文件职责保持不变。

1. 确认项目根目录内存在 `frontend/`、`backend/`、`docs/`。
2. 在 `frontend/` 执行依赖安装并确认可以运行 `npm run dev` 与 `npm run build`。
3. 启动 Flask 后端，依次请求：
   - `GET /api/kpi?year=2024`
   - `GET /api/monthly?year=2024`
   - `GET /api/zones?year=2024`
   - `GET /api/ranking?year=2024&category=hottest&limit=15`
4. 确认每个成功响应的 `data` 是数组；不要根据静态稿的示例字段假设接口返回城市、国家或历史基线。
5. 检查当前工作区是否有用户未提交改动。只修改本手册列出的文件，避免覆盖无关改动。

### 2.1 数据能力对照表（必须先读）

| 设计元素 | 当前可用数据 | 实现决定 |
|---|---|---|
| 四张 KPI 卡 | `getKPI(year)` | 完整实现，附属描述使用固定说明或 KPI `kpi_desc`；不显示虚构同比和运行率。 |
| 月度均温/均高/均低折线 | `getMonthly(year)` | 完整实现。 |
| 气候带占比 | `getZones(year)` | 完整实现。 |
| 四类站点 TOP 15 | `getRanking(year, category, 15)` | 完整实现；展示“站点”而非错误称为“城市”。 |
| 排名明细 | `rank_num, station_name, station_id, value` | 表格仅展示这四种真实字段和单位。 |
| 区域/半球/气候带筛选趋势 | 无 | 第一版不展示可操作筛选；若要保留视觉控件，应禁用并标明“接口暂未接入”。 |
| 1990–2020 基准、异常值、置信区间 | 无 | 第一版不展示或显示“暂无基准数据”，不能计算假值。 |
| 全屏图表 | 前端能力 | 完整实现。 |
| 导出当前图表 | 前端能力 | 可选实现，调用 ECharts `getDataURL()`；不是导出原始数据。 |
| 国家、最高记录温度、热天数 | 无 | 第一版不展示。 |
| AI 助手问答 | `askAgent(question, year)` | 完整保留；仅做视觉重构。 |
| 告警、历史、设置、语言、账户 | 无 | 不创建可点击但无功能的入口。 |

## 4. 先建立全局视觉系统

### 4.1 `styles/tokens.css`

创建以下 CSS 变量，所有页面只能引用变量，禁止在各个页面散落同一组十六进制颜色。

```css
:root {
  --ci-primary: #14422d;
  --ci-primary-container: #2d5a43;
  --ci-primary-soft: #bceecf;
  --ci-green: #3a674f;
  --ci-secondary: #39656b;
  --ci-secondary-soft: #bae8ef;
  --ci-tertiary: #8b3713;
  --ci-tertiary-soft: #ffdbce;
  --ci-error: #ba1a1a;
  --ci-surface: #faf9f7;
  --ci-surface-lowest: #ffffff;
  --ci-surface-container: #efeeec;
  --ci-surface-variant: #e3e2e0;
  --ci-text: #1a1c1b;
  --ci-text-muted: #414943;
  --ci-outline: #717973;
  --ci-outline-variant: #c0c9c1;
  --ci-glass: rgb(255 255 255 / 85%);
  --ci-glass-strong: rgb(250 249 247 / 90%);
  --ci-shadow: 0 4px 20px rgb(0 0 0 / 5%);
  --ci-radius-sm: 8px;
  --ci-radius-md: 12px;
  --ci-radius-lg: 16px;
  --ci-content-max: 1200px;
  --ci-page-gutter: 40px;
  --ci-gap: 24px;
}

@media (max-width: 767px) {
  :root { --ci-page-gutter: 20px; --ci-gap: 16px; }
}
```

### 4.2 `styles/base.css`

实现以下规则。

1. `html, body, #app` 高度至少为 `100%`；`body` 使用系统中文字体优先级：`Inter, "Microsoft YaHei", "PingFang SC", sans-serif`。
2. 为页面根容器 `.app-shell` 设置 `min-height: 100vh`、文字色 `--ci-text`、背景色 `--ci-surface`。
3. 若有授权图片，使用伪元素或独立 `.app-background` 固定铺满，`object-fit: cover`；再覆盖一层 `rgb(250 249 247 / 78%)` + `backdrop-filter: blur(2px)`。背景层必须 `pointer-events: none; z-index: -1`。
4. 无图片时使用备用背景：

```css
background:
  radial-gradient(circle at 5% 0%, rgb(188 238 207 / 55%), transparent 34rem),
  radial-gradient(circle at 98% 20%, rgb(189 234 242 / 42%), transparent 30rem),
  var(--ci-surface);
```

5. 统一键盘焦点：`outline: 3px solid rgb(57 101 107 / 45%); outline-offset: 2px;`。不得只依靠 hover 表示可点击状态。
6. 在 `@media (prefers-reduced-motion: reduce)` 中禁用过渡、脉冲和浮动动画。

### 4.3 公共组件行为

**`GlassCard.vue`**：根元素提供 `glass-card` 类，使用 `background: var(--ci-glass)`、`backdrop-filter: blur(12px)`、1px 半透明边框、`var(--ci-shadow)`、`var(--ci-radius-lg)`；通过默认 slot 渲染内容。组件不负责高度和网格布局。

**`ChartPanel.vue`**：接收 `title`、`subtitle`、`loading`、`empty`、`emptyText` 属性；提供 `actions` 和默认 slot。图表容器最小高度为 320px，桌面版可以由父级指定 450px 或 600px。没有数据时在面板内部而不是替换整个页面显示 `el-empty`。

**`PageState.vue`**：接收 `loading`、`error`、`empty`、`emptyText`，并向外抛出 `retry`。逻辑顺序必须为 loading → error → empty → default slot。所有路由页统一使用，避免四种页面各自写一套状态分支。

### 4.4 ECharts 视觉工具

在 `useDashboardTheme.js` 导出以下不可变常量：

```js
export const chartColors = {
  green: '#3a674f', teal: '#39656b', orange: '#8b3713',
  orangeSoft: '#ffdbce', error: '#ba1a1a', text: '#414943',
  outline: '#c0c9c1', grid: '#e3e2e0', primary: '#14422d',
}
```

并封装 `baseTooltip()`、`baseGrid()`、`monthLabels()`、`safeNumber()`。`safeNumber(value)` 对 `null`、空字符串、非有限数字返回 `null`，不可使用 `Number(value) || 0`，否则缺失值会被伪装成 0。

Tooltip 统一要求：白色 95% 背景、边框 `outline`、深色文本、阴影；数值格式化为最多两位小数，但仅在数据是数字时调用 `toFixed`。圆环图 tooltip 显示站点数和百分比。

## 5. 应用骨架：按顺序改 `main.js`、`App.vue`

### 步骤 1：`main.js`

在创建应用之前导入：

```js
import './styles/tokens.css'
import './styles/base.css'
```

保留 Element Plus、路由和 ECharts 按需注册的现有实现。不要额外注册全局 ECharts 实例。

### 步骤 2：`App.vue`

1. 保留 `availableYears = [2022, 2023, 2024]`、`selectedYear = ref(2024)` 与 `provide('selectedYear', selectedYear)`。
2. 将目前内联的顶部导航替换为 `<AppTopNav v-model:year="selectedYear" :available-years="availableYears" />`。若不使用双向绑定，则改用 `@update:year="selectedYear = $event"`，但只保留一种写法。
3. 外层使用：

```vue
<div class="app-shell">
  <div class="app-background" aria-hidden="true" />
  <AppTopNav ... />
  <main class="app-main"><router-view /></main>
  <AppFooter />
  <BIAgent />
</div>
```

4. `app-main` 顶部预留 80px 导航高度；内容宽度由各页面的 `.page-container` 控制，不在 `App.vue` 强制 1200px。
5. **删除** `router-view` 上的 `:key="$route.path + selectedYear"`。每个页面在 `watch(selectedYear, load, { immediate: true })` 中加载数据即可。保留两种机制会导致一年切换时组件销毁重建后又触发 watcher，出现重复请求。
6. 顶部导航的“设置、语言、账户”不在本次范围内，不能显示为可点击图标。右侧仅保留年份 `el-select`；窄屏可显示为图标触发的年份选择，但必须实际可用。

### 步骤 3：公共导航与页脚

`AppTopNav.vue` 要求：固定顶部、高 64px（移动端与桌面端一致）、半透明背景、毛玻璃、底边框；左侧品牌名；中部 `router-link`：总览、趋势、排名、气候带；右侧年份选择器。使用路由 `exact-active-class` 或 CSS `.router-link-active` 显示绿色文字和 2px 主色底边。

小于 768px 时：隐藏中部导航，展示一个真实的“菜单”按钮打开 `el-drawer` 或 `el-popover` 中的同一组路由链接。禁止直接隐藏导航而使其他页面不可达。

`AppFooter.vue`：包含版权和“数据来源：NOAA GSOD”；“方法说明、API 访问、联系支持”若没有真实链接，渲染为普通文本而不是 `href="#"`。桌面横向、移动端纵向居中。

## 6. 页面一：`Dashboard.vue`（总览）

### 6.1 数据加载

保留一次并发加载：

```js
const [kpiRes, monthlyRes, zonesRes, rankingRes] = await Promise.all([
  getKPI(selectedYear.value),
  getMonthly(selectedYear.value),
  getZones(selectedYear.value),
  getRanking(selectedYear.value, 'hottest', 15),
])
```

请求开始前重置 `loading=true`、`error=''`、`empty=false`；四项结果都处理完再关闭 loading。组件卸载或年份快速切换时，用递增 `requestId` 防止旧请求覆盖新年份结果：开始时保存 `const id = ++requestId`，每次写入响应式状态前检查 `id === requestId`。

KPI 映射保持既有字段：

| API `kpi_name` | 界面标题 | 单位 |
|---|---|---|
| `global_avg_temp` | 全球年平均气温 | °C |
| `total_stations` | 活跃气象站 | 无 |
| `extreme_event_pct` | 极端天气占比 | % |
| `hottest_station_temp` | 年度最高温 | °C |

`kpi_value` 为 `0` 时必须显示 `0`，不能被 `|| '--'` 覆盖。使用 `value == null ? '--' : value`。

### 6.2 模板布局

页面容器：`max-width: var(--ci-content-max); margin: 0 auto; padding: 48px var(--ci-page-gutter) 80px;`。

正常状态下从上到下：

1. H1：`{{ selectedYear }} 年全球气候总览`，桌面 48px/600，移动端 32px。
2. 四张 KPI 玻璃卡：CSS Grid，`repeat(4, minmax(0, 1fr))`；1024px 以下两列，640px 以下一列；每张内边距 24px，最小高度 180px。
3. KPI 卡内容：上方小写/字距加宽标签、状态圆点、数值；最下方只显示真实描述（如 `kpi_desc`）或固定数据口径“基于 NOAA GSOD 年度汇总”。不得显示设计稿的虚构“近十年 +0.4°C”“98% operational”“Death Valley”。
4. 中间图表网格：桌面 `8fr 4fr`，小于 1024px 改为一列。左侧月度趋势高度 450px，右侧气候带高度 450px。
5. 底部 TOP 15 面板，高度最小 400px。

### 6.3 月度趋势 ECharts

从 `monthly` 按 `obs_month` 升序排序；`xAxis.data` 固定为 `1月` 到 `12月`，缺失月份对应 `null`。三条线：

| 名称 | 字段 | 颜色 | 样式 |
|---|---|---|---|
| 平均气温 | `avg_temp` | `green` | 平滑、8px 圆点、浅绿色面积渐变 |
| 平均最高气温 | `avg_max` | `orange` | 平滑、8px 圆点、浅橙面积渐变 |
| 平均最低气温 | `avg_min` | `teal` | 平滑、8px 圆点、无面积或浅青面积 |

轴和网格按 `useDashboardTheme.js`；Y 轴名称为 `°C`；`boundaryGap: false`；legend 在顶部。图表右上角保留“全屏”和“下载图片”按钮：全屏调用 `chartRef.value?.getEchartsInstance()?.getDom().requestFullscreen()`，下载用实例 `getDataURL({ pixelRatio: 2, backgroundColor: '#faf9f7' })` 生成临时链接。任何浏览器不支持时仅提示 `ElMessage.info`，不报错。

### 6.4 气候带环形图

数据映射：`tropical → 热带`、`temperate → 温带`、`continental → 大陆性`、`polar → 寒带`、`arid → 干旱`。

颜色强制为：热带 `#8b3713`、温带 `#3a674f`、干旱 `#bdeaf2`、大陆性 `#39656b`、寒带 `#c0c9c1`。使用 `radius: ['50%', '75%']`、白色 3px 分隔、图例底部水平排列、默认不显示标签；hover 时显示中心标签。若 `zones` 数据为空，只在该卡内显示 `暂无气候带数据`，不得使整页失败。

### 6.5 TOP 15 图表

按设计稿改为**水平条形图**，而非现有纵向条形图：

- `xAxis.type = 'value'`，单位 `°C`；`yAxis.type = 'category'`。
- 按数值升序排列，使最高值位于最上方（ECharts category 轴可反转或对数组处理，渲染结果必须正确）。
- 站名使用 `stationCN(name)`，超过 22 个字符省略；完整名称放在 axis label tooltip 或图表 tooltip。
- 条形使用从 `#ffdbce` 到 `#8b3713` 的水平渐变，`borderRadius: [0, 6, 6, 0]`。
- 标题为“最高温站点 TOP 15”，副标题为“按年度站点最高温排序”。不要称为“城市”，因为 API 返回的是站点。

### 6.6 状态与验收

整页 loading 使用 `PageState` 骨架屏；全局错误时显示重试；当 KPI 和月度数据均为空时显示整页空状态。区域图或榜单缺失只降级各自卡片。切换年份后，标题、四张卡、三张图均更新，且网络面板中每个端点仅请求一次。

## 7. 页面二：`CityRanking.vue`（排名）

### 7.1 状态和交互

保留并明确以下状态：

```js
const category = ref('hottest')
const categoryMeta = {
  hottest: { label: '最高温', unit: '°C', title: '最高温站点 TOP 15' },
  coldest: { label: '最低温', unit: '°C', title: '最低温站点 TOP 15' },
  rainiest: { label: '最多降水', unit: 'mm', title: '最多降水站点 TOP 15' },
  most_extreme: { label: '极端天气', unit: '天', title: '极端天气站点 TOP 15' },
}
```

类别切换应使用语义化 `el-radio-group` 或四个 `button` 组成的 `role="tablist"`；选中态为主色背景白字，未选中态为深灰文字透明背景。点击后仅请求 `getRanking(selectedYear, category, 15)`，不得重新请求 Dashboard 的其他接口。

### 7.2 页面布局

1. 顶部标题：`{{ selectedYear }} 年全球站点排名`；副标题“基于 NOAA GSOD 年度聚合数据”。不要使用“每 24 小时更新”。
2. 标题同一行右侧放类别分段控件；移动端控件可以横向滚动，不能压缩文字或换成不可见内容。
3. 主图使用玻璃卡，最小高度 540px。标题左侧有一个与类别相匹配的状态点，右侧显示真实单位。
4. 下方为明细表玻璃卡。设计稿中的“Load Remaining Rows”不实现：当前接口明确最多 TOP15；表格一次展示接口返回的全部行。

### 7.3 图表

排名页设计稿是纵向柱状图，按此页实现：

- `xAxis.type = 'category'`，站点名截断为 12 个可见字符，标签旋转 25°；完整名由 tooltip 显示。
- `yAxis.type = 'value'`，名称使用 `categoryMeta[category].unit`。
- `series.type = 'bar'`，`barMaxWidth: 44`，从 `primary-container` 到 `secondary` 的渐变；前三名可用三个深浅层级，但不得改变数值含义。
- tooltip 包含排名、站点全名、数值和单位；数值保留两位小数。
- 图表数据按 `rank_num` 升序，不能依赖服务端默认顺序。

### 7.4 明细表（严格按真实字段）

用 Element Plus `el-table` 替换静态 HTML 表格。第一版列为：排名、站点名称、站点 ID、指标值。最后一列标题和单位随 `category` 改变。

不要实现设计稿中的“城市、国家、平均最高温、最高记录、35°C 以上天数”列，除非后端新接口在第 11 节完成。`stationCN()` 仅用于展示名称；没有中文映射时保留原始站名。

给 `el-table` 加 `row-class-name` 或 CSS，使 hover 使用半透明 `surface-variant`；第一名的排名文字使用 `--ci-tertiary`。小屏时表格允许横向滚动，不能挤压或丢失列。

## 8. 页面三：`TrendAnalysis.vue`（趋势）

### 8.1 第一版必须实现的真实范围

当前 API 仅返回一个年份的全局 `avg_temp`、`avg_max`、`avg_min`，不能支持设计稿中的“区域筛选”“1990–2020 历史均线”“温度异常”“置信区间”。本页第一版的真实功能是：展示已选年份全球月度气温趋势。

因此，**不要**把静态稿中的复选框和“Apply Filters”按钮以可交互状态上线。推荐的第一版布局：

- 左侧 280px 信息栏保留，第一张卡展示“数据范围”与只读值“全球 / {{ selectedYear }} 年”；第二张卡展示“年度峰值平均气温”，由月度 `avg_temp` 的最大值计算，并显示月份。
- 主区域标题改为“月度气温变化”；副标题改为“全球月度平均、平均最高与平均最低气温”。
- 右上角保留真实可用的全屏与下载图片操作。
- 下方三张小卡展示：全年平均气温（`avg_temp` 均值）、年度最高平均气温（`avg_max` 最大值）、数据来源（`NOAA GSOD`）。不用显示“置信区间”。

### 8.2 数据计算

加载逻辑仅调用 `getMonthly(selectedYear.value)`，排序规则同 Dashboard。计算函数必须忽略 `null`、`undefined`、非有限数值：

```js
const valid = rows.map(row => Number(row.avg_temp)).filter(Number.isFinite)
const annualAverage = valid.length ? valid.reduce((a, b) => a + b, 0) / valid.length : null
```

峰值月份从有效的 `avg_temp` 中取最大值对应行，文字格式为“{{ peakMonth }} 月记录”。没有有效数据时显示 `--` 和“暂无数据”。

### 8.3 图表

主图高度固定 600px（移动端 420px），使用 Dashboard 同一组三线数据和颜色；与总览的区别仅为：更大的容器、`axisPointer: { type: 'cross' }`、详细副标题、全屏/导出按钮。Y 轴名称 `气温（°C）`，不要命名为“异常值（°C）”。

若后端将来提供基准 API，在同一图内新增一条深绿实线“历史基准”，并将表单变为可操作；在此之前不得使用任意硬编码基准数组。

## 14. 常见错误与禁止事项

1. **错误：复制静态稿 HTML。** 这会绕开 Vue 生命周期、现有 API、路由和 `vue-echarts`，禁止。
2. **错误：给每个页面单独定义一套主题颜色。** 必须使用 `tokens.css`。
3. **错误：保留 `router-view` 的年份 key 再同时 watch 年份。** 会重复加载，必须删除 key。
4. **错误：`Number(value) || 0`。** 会把缺失数据变成 0，导致图表和 KPI 误导用户。
5. **错误：把 `station_name` 标为“城市”。** 数据源是气象站，除非后端新增 city 字段。
6. **错误：将“设置、告警、历史、导出”做成没有后端能力的假按钮。** 移除或在产品确认后实现。
7. **错误：使用外部背景 URL。** 必须用受控资源或 CSS 备用背景。
8. **错误：仅在大屏调样式。** 每个改动都要在 375px、768px、1024px、1440px 四种宽度检查。

完成以上要求，即可在不改变现有真实数据能力的前提下，将静态设计稿可靠地落地为可维护的 ClimateInsight Vue 前端。
