# 实习日志 — 2026年7月21日

今天对 ClimateInsight 全球气候智能分析平台进行了全面的前端视觉改版与AI助手架构升级。

## 一、前端视觉全面重构

按照设计稿规范，将整个前端从深蓝科技风改造为 Biophilic Harmony 自然有机风格。建立了完整的设计令牌体系（CSS变量），包括森林绿主色（#14422d）、青绿辅助色（#39656b）、陶土橙点缀色（#8b3713）和暖白底色（#faf9f7）。所有页面统一采用毛玻璃卡片、自然风景背景和柔和阴影。

新增了 AppTopNav（顶部导航栏）、AppSidebar（左侧年份选择栏）、AppFooter（页脚）、GlassCard（毛玻璃容器）、ChartPanel（图表面板）、PageState（统一三态处理）六个公共组件，实现了 loading/error/empty 状态的标准化处理。创建了 useDashboardTheme 和 useChartActions 两个工具模块，统一管理 ECharts 图表主题配置和操作。

## 二、AI助手架构升级

将AI助手的意图识别从纯规则引擎升级为大模型驱动的智能调度架构。通过 DeepSeek API，LLM 仅输出结构化 JSON 意图（kpi/monthly/ranking/zones/chat/help），经 Schema 校验后调用白名单工具执行查询，任何环节失败自动回退规则引擎。三层安全防护确保了系统的稳定性。

实现了跨年查询与多年对比功能。用户在任意年份页面均可查询其他年份数据，LLM 自动识别问题中的年份并覆盖 UI 默认值。"2022和2023年哪个更热"等对比问题可生成柱状图对比。

AI助手的交互形态从悬浮可拖拽窗口改为右侧固定分屏面板（320px），展开时内容区自动收缩适配，收回时显示圆形悬浮按钮。解决了 CSS Grid 中 min-width:0 导致的宽度自适应问题。

## 三、数据可视化优化

排名页面柱状图实现了按数值映射单色渐变——最高温从浅黄到深红、最低温从深蓝到浅蓝，直观区分极端程度。Dashboard 图表改为水平条形图，Tooltip 统一保留两位小数。

## 四、问题与收获

全天最大的技术难点是 AI 面板展开后图表不自适应的问题。经过反复排查，最终定位到 CSS Grid 布局中 auto 列的 min-width:auto 默认行为导致列宽无法收缩，设置 min-width:0 后解决。这加深了对 CSS Grid 弹性布局机制的理解。
