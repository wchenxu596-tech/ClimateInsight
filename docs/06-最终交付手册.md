## 2. 系统架构

```
NOAA GSOD tar.gz (~100MB/年)
  → scripts/download_and_load.py (下载→解压→清洗→聚合)
    → MySQL DWS: dws_station_monthly (153万行, 站×月)
      → MySQL ADS: 6张预聚合表 (极速查询 0.01–0.15s)
        → Flask API: 14 REST端点 + @cached(300s)
          → Vue 3 前端: 6页面 + AI面板 + Grainient背景
```

**技术栈**：Python 3.11 + Flask 3.1 + MySQL 8.0 + Vue 3.5 + ECharts 6 + Docker

## 5. 数据库

### 5.1 表清单

| 表 | 层级 | 行数 | 用途 |
|---|---|---|---|
| `dws_station_monthly` | DWS | 1,530,000 | 站×月明细（21 字段） |
| `ads_kpi` | ADS | 44 | 全球 KPI（4 指标 × 11 年） |
| `ads_monthly_trend` | ADS | ~130 | 月度温度 |
| `ads_ranking` | ADS | ~660 | 4 类排名 TOP15 |
| `ads_zones` | ADS | 55 | 5 气候带站点数 |
| `ads_zone_trends` | ADS | 55 | 气候带温度/降水/极端 |
| `ads_stations` | ADS | 133,954 | 站点概要（15 列，含 6 类极端事件） |

### 5.2 关键索引

- `PRIMARY KEY (station_id, year, obs_month)` — DWS 主键
- `idx_year_sid (year, station_id)` — 站级 GROUP BY 加速
- `idx_climate_zone (year, climate_zone)` — 气候带聚合加速
- 所有 ADS 表均含 `PRIMARY KEY (data_year, ...)`

## 6. 安全措施

| 层级 | 措施 |
|---|---|
| SQL 注入 | 100% 参数化查询（`%s` 占位符，零字符串拼接） |
| 输入拦截 | SQL 关键词黑名单（DROP/DELETE/SELECT 等） |
| 数据集白名单 | DatasetCatalog 仅允许 ADS 表访问 |
| 意图校验 | JSON Schema 校验 + 多余字段拒绝 |
| 年份范围 | 白名单 `{2015..2025}` |
| LLM 安全 | temperature=0，输出不直接执行，Schema 约束 |
| 前端渲染 | `{{ }}` 插值，禁止 `v-html` |

## 7. 部署

```bash
git clone https://github.com/wchenxu596-tech/ClimateInsight.git
cd ClimateInsight
cp .env.example .env
docker compose up -d
# 访问 http://localhost:8080
```

数据导入：
```bash
PYTHONIOENCODING=utf-8 python scripts/download_and_load.py --year 2025
python scripts/import_all_years.py  # 批量导入所有年份
```
