# ClimateInsight 部署运行手册

> 最后更新: 2026-07-24
> 项目名称: ClimateInsight -- 基于 NOAA GSOD 数据的全球气候智能分析平台
> 数据范围: 2015--2025（11 年）

---

## 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| Docker | 20.10+ | 推荐使用 Docker Desktop |
| Docker Compose | 2.0+ | 内置在 Docker Desktop 中 |
| Python | 3.11+ | 仅本地数据导入时需要 |
| Node.js | 20+ | 仅前端本地开发和构建时需要 |

---

## 快速启动 (Docker)

推荐方式，一条命令启动全部服务。

```bash
# 1. 克隆仓库
git clone https://github.com/wchenxu596-tech/ClimateInsight.git
cd ClimateInsight

# 2. 配置环境变量（首次或重置时）
cp .env.example .env
# 可选：编辑 .env 修改密码、端口、AI Agent 等配置

# 3. 启动全部服务
docker compose up -d

# 4. 验证服务状态
docker compose ps
curl http://localhost:5000/api/health

# 5. 浏览器访问
# http://localhost:8080
```

首次启动会自动执行以下操作：
- MySQL 容器初始化，执行 `sql/mysql/001_init.sql` 建表和 `sql/mysql/002_create_users.sql` 创建应用用户
- Backend 等待 MySQL 就绪后启动 Flask API 服务
- Frontend 等待 Backend 就绪后启动 Nginx 静态服务

注意：Docker 部署不包含数据导入。数据需要通过下面的"数据导入"步骤单独加载。

---

## 服务说明

Docker Compose 启动的服务一览：

| 服务 | 容器名 | 端口映射 | 技术栈 | 说明 |
|------|--------|----------|--------|------|
| mysql | climate_mysql | 3307:3306 | MySQL 8.0 | 命名卷 `mysql_data` 持久化，数据库 `climate_dw` |
| backend | climate_backend | 5000:5000 | Flask + gunicorn (2 workers) | RESTful API，支持 CORS，可启用 AI Agent |
| frontend | climate_frontend | 8080:80 | Nginx + Vue 3 SPA | API 反向代理 `/api/*` -> backend:5000，SPA fallback |
| setup-data | climate_setup_data | -- | Python 3.11 | 一次性数据加载任务，通过 `profile: setup` 手动运行 |

数据卷：

| 卷名 | 用途 |
|------|------|
| mysql_data | MySQL 数据持久化 |
| climate_data | NOAA GSOD 原始下载数据缓存 |

## 数据导入

### 方式一：单年份导入

从 NOAA GSOD 官网下载原始数据、预处理、聚合后写入 MySQL，全自动完成。

```bash
# 导入单一年份（Windows 需要设置 PYTHONIOENCODING 避免 GBK 编码问题）
PYTHONIOENCODING=utf-8 python scripts/download_and_load.py --year 2025

# 跳过下载，使用已有的 tar.gz（需要提前放到 data/raw/ 目录）
PYTHONIOENCODING=utf-8 python scripts/download_and_load.py --year 2025 --skip-download
```

脚本执行步骤：
1. 从 `https://www.ncei.noaa.gov/data/global-summary-of-the-day/archive/{year}.tar.gz` 下载年度归档（约 500MB）
2. 解压并合并所有气象站 CSV 为单个文件
3. 按月聚合（DWS 层）并计算 KPI / 排名 / 趋势 / 气候带（ADS 层）
4. 写入 MySQL 对应表

### 方式二：批量导入全部年份

前提：已手动下载 NOAA GSOD 年度 tar.gz 文件到 `data/raw/` 目录。

```bash
python scripts/import_all_years.py
```

脚本会自动发现 `data/raw/` 中已存在的 `.tar.gz` 文件，逐个调用 `download_and_load.py --skip-download` 导入。缺失的年份会打印出下载 URL 提示手动下载。

### 数据校验

导入完成后建议运行校验脚本检查数据完整性。

```bash
MYSQL_PASSWORD=climate123 python scripts/verify_data.py --year 2025
```

校验项目包括：
- DWS 表行数是否在合理范围
- 主键唯一性（无重复）
- 温度范围合法性（-90~60 度）
- KPI 年均温交叉验证
- 月度趋势是否覆盖 12 个月
- 排名数据每类是否 15 条
- 气候带分类是否覆盖

## 本地开发

适用于需要修改代码的开发场景。Docker 仅运行 MySQL，前后端在宿主机运行。

### 1. 启动 MySQL

```bash
docker compose up -d mysql
# MySQL 暴露在 127.0.0.1:3307，root 密码见 .env 中的 MYSQL_ROOT_PASSWORD
```

### 2. 导入数据（首次）

```bash
# Windows
$env:MYSQL_HOST='127.0.0.1'
$env:MYSQL_PORT='3307'
$env:MYSQL_USER='root'
$env:MYSQL_PASSWORD='climate123'

PYTHONIOENCODING=utf-8 python scripts/download_and_load.py --year 2025
```

### 3. 启动后端

```bash
cd backend
pip install -r requirements.txt
$env:MYSQL_HOST='127.0.0.1'
$env:MYSQL_PORT='3307'
$env:MYSQL_USER='root'
$env:MYSQL_PASSWORD='climate123'
python app.py
# 后端运行在 http://localhost:5000
```

### 4. 启动前端（新终端）

```bash
cd frontend
npm install
npm run dev
# 前端运行在 http://localhost:5173（Vite 开发服务器）
```

前端开发服务器需要通过 Vite 配置代理 `/api/*` 到 `http://localhost:5000`，否则 API 请求会失败。

## 生产构建

### 前端构建

```bash
cd frontend
npm ci
npm run build
# 产物在 dist/ 目录，可直接部署到任意静态服务器
```

### 后端生产运行

```bash
cd backend
gunicorn --bind 0.0.0.0:5000 --workers 4 --timeout 60 app:app
```

### Nginx 反向代理（推荐）

将编译好的前端 `dist/` 目录部署到 Nginx，并配置 `/api/*` 反向代理到后端。参考 `frontend/nginx.conf` 模板：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:5000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_read_timeout 60s;
}

location / {
    root /path/to/frontend/dist;
    try_files $uri $uri/ /index.html;
}
```

## .env 配置参考

完整的 `.env` 配置项说明（从 `.env.example` 复制后修改）。

```ini
# ─── MySQL ───
MYSQL_HOST=mysql             # 数据库地址（Docker 内用服务名 mysql，本地用 127.0.0.1）
MYSQL_PORT=3306              # 数据库端口（Docker 内用 3306，本地用 3307）
MYSQL_DATABASE=climate_dw    # 数据库名
MYSQL_USER=climate_app       # 应用用户（读写），Docker 初始化时自动创建
MYSQL_PASSWORD=climate123    # 应用用户密码
MYSQL_ROOT_PASSWORD=climate123  # MySQL root 密码

# ─── 端口 ───
API_PORT=5000                # 后端 API 端口
FRONTEND_PORT=8080           # 前端访问端口
MYSQL_PORT=3307              # 内置 MySQL 宿主机映射端口（避免与本地 MySQL 冲突）

# ─── Flask ───
FLASK_DEBUG=false            # 开发模式（生产必须设为 false）
CORS_ORIGINS=http://localhost:8080,http://localhost:5173  # 允许的跨域来源

# ─── AI Agent（可选，留空则使用纯规则引擎） ───
AGENT_ENABLED=false          # 是否启用 AI 分析助手
LLM_BASE_URL=                # LLM API 地址（兼容 OpenAI 格式）
LLM_API_KEY=                 # LLM API 密钥
LLM_MODEL=                   # 模型名称（如 gpt-4o, deepseek-chat 等）
```

## 常用命令

```bash
# 查看所有服务状态
docker compose ps

# 查看日志
docker compose logs -f backend
docker compose logs -f frontend
docker compose logs -f --tail=100 mysql

# 重启单个服务
docker compose restart backend

# 进入 MySQL 命令行
docker exec -it climate_mysql mysql -u root -p climate_dw

# 停止所有服务（保留数据卷）
docker compose down

# 停止所有服务并删除数据卷（重置数据库，数据将丢失）
docker compose down -v

# 重新构建并启动
docker compose up -d --build

# 手动运行数据加载（profile: setup）
docker compose run --rm setup-data python /scripts/download_and_load.py --year 2025

# 检查后端健康状态
curl http://localhost:5000/api/health

# 查看 MySQL 数据
docker exec -it climate_mysql mysql -u root -pclimate123 -e \
  "SELECT kpi_name, kpi_value, data_year FROM climate_dw.ads_kpi ORDER BY data_year DESC, kpi_name"
```

## 故障排查

**Q: 启动后访问 localhost:8080 页面空白**

A: 检查所有服务状态：`docker compose ps`。如果 `climate_frontend` 状态为 `unhealthy` 或 `Exited`，可能是因为 `climate_backend` 未就绪。等待 backend 健康后再重启 frontend：`docker compose restart frontend`。

**Q: MySQL 端口冲突（Port 3307 already in use）**

A: 端口 3307 已被本地其他 MySQL 实例占用。在 `.env` 中修改 `MYSQL_PORT=3308`（或其他空闲端口），然后重启：`docker compose down && docker compose up -d`。

**Q: 前端端口冲突（Port 8080 already in use）**

A: 在 `.env` 中修改 `FRONTEND_PORT=8081`，重启后访问 `http://localhost:8081`。

**Q: 后端端口冲突（Port 5000 already in use）**

A: 在 `.env` 中修改 `API_PORT=5001`，同时更新 `CORS_ORIGINS` 中对应的端口号。

**Q: 数据导入时 Windows 下出现 GBK 编码错误**

A: 在命令前加 `PYTHONIOENCODING=utf-8`：

```bash
PYTHONIOENCODING=utf-8 python scripts/download_and_load.py --year 2025
```

**Q: MySQL 连接失败 (2003, Can't connect)**

A: 检查以下几点：
1. `docker compose ps` 确认 climate_mysql 状态为 `healthy`
2. 本地开发时 MySQL 端口为 3307（不是 3306），检查 `MYSQL_PORT` 环境变量
3. 等待 MySQL 容器完全启动（首次启动约需 30 秒）

**Q: AI 分析助手无响应**

A: AI Agent 默认关闭，需要在 `.env` 中配置：
1. `AGENT_ENABLED=true`
2. `LLM_BASE_URL` -- LLM API 地址
3. `LLM_API_KEY` -- API 密钥
4. `LLM_MODEL` -- 模型名称
5. 配置后重启：`docker compose restart backend`

**Q: 地图页面加载缓慢或数据为空**

A: 检查 `ads_stations` 和 `ads_zone_trends` 表是否有数据：

```bash
docker exec -it climate_mysql mysql -u root -pclimate123 -e \
  "SELECT data_year, COUNT(*) FROM climate_dw.ads_stations GROUP BY data_year"
```

如果没有数据，需要运行数据导入脚本。

**Q: 需要添加新年份的数据列或索引**

A: 查看 `sql/mysql/` 目录下的增量升级脚本：
- `sql/mysql/002_add_dws_weather_days.sql` -- DWS 表新增天气日数相关列
- `sql/mysql/003_ads_year_keys.sql` -- ADS 表复合主键升级

按顺序执行：`docker exec -i climate_mysql mysql -u root -pclimate123 climate_dw < sql/mysql/xxx.sql`

**Q: 完全重置项目**

```bash
docker compose down -v        # 停止并删除所有数据
docker compose up -d          # 重新启动（自动初始化数据库）
PYTHONIOENCODING=utf-8 python scripts/download_and_load.py --year 2025  # 重新导入数据
```

**Q: 如何查看某个年份导入了多少条数据**

```bash
docker exec -it climate_mysql mysql -u root -pclimate123 -e \
  "SELECT year, COUNT(*) as stations, SUM(obs_days) as records
   FROM climate_dw.dws_station_monthly GROUP BY year ORDER BY year"
```
