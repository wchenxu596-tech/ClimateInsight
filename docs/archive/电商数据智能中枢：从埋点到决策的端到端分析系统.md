## 一、项目概述

### 1.1 项目背景

某大型电商平台日均产生数千万级用户行为数据，传统BI报表开发周期长、灵活性差，业务部门需要数天才能获取分析结果。本项目构建一套**企业级数据仓库 + 智能BI分析平台**，实现数据自动化入仓、多维分析、可视化大屏、智能问答和报表自动生成，将数据分析周期从天级缩短至分钟级。

### 1.2 项目目标

| 目标             | 指标                           |
| ---------------- | ------------------------------ |
| 数据仓库分层建模 | ODS→DWD→DWS→ADS四层架构        |
| 用户价值分析     | RFM模型自动分群                |
| 商品关联分析     | 购物篮关联规则挖掘             |
| 可视化大屏       | 5+分析页面，支持实时切换       |
| 智能BI问答       | NL2SQL自然语言查询，准确率≥85% |
| 智能报表生成     | Agent自动读取数据生成分析日报  |

### 1.3 数据集

**天池淘宝用户行为数据集**（[下载链接](https://tianchi.aliyun.com/dataset/649)）：
- **规模**：100万+用户、400万+商品、1亿+行为记录
- **时间跨度**：2017年11月25日 ~ 2017年12月3日（9天）
- **行为类型**：点击(pv)、加购(cart)、收藏(fav)、购买(buy)
- **字段结构**：

| 字段          | 说明                        |
| ------------- | --------------------------- |
| user_id       | 用户ID（脱敏）              |
| item_id       | 商品ID（脱敏）              |
| category_id   | 商品类目ID（脱敏）          |
| behavior_type | 行为类型（pv/cart/fav/buy） |
| timestamp     | 行为时间戳                  |

#### DWD层 - 数据明细层

```sql
-- ============================================
-- DWD层：用户行为明细事实表（清洗+标准化）
-- ============================================
CREATE TABLE IF NOT EXISTS dwd_user_behavior_fact (
    -- 行为ID（生成唯一标识）
    behavior_id     STRING COMMENT '行为唯一ID: md5(user_id+item_id+ts+behavior_type)',
    
    -- 维度外键
    user_id         STRING COMMENT '用户ID',
    item_id         STRING COMMENT '商品ID',
    category_id     STRING COMMENT '商品类目ID',
    
    -- 行为属性
    behavior_type   STRING COMMENT '行为类型: page_view(浏览)/add_cart(加购)/add_fav(收藏)/purchase(购买)',
    behavior_time   TIMESTAMP COMMENT '行为时间',
    
    -- 时间维度退化（冗余常用时间字段，减少JOIN）
    behavior_date   STRING COMMENT '行为日期: yyyy-MM-dd',
    behavior_hour   INT COMMENT '行为小时: 0-23',
    behavior_weekday INT COMMENT '星期几: 1-7',
    is_weekend      INT COMMENT '是否周末: 0否/1是',
    
    -- 数据质量标记
    data_quality_flag STRING COMMENT '数据质量标记: valid(有效)/invalid(无效)'
) 
COMMENT '用户行为明细事实表'
PARTITIONED BY (dt STRING)
STORED AS ORC
LOCATION '/warehouse/dwd/dwd_user_behavior_fact';

-- DWD层ETL清洗逻辑
INSERT OVERWRITE TABLE dwd_user_behavior_fact PARTITION(dt='${biz_date}')
SELECT 
    md5(concat(user_id, item_id, ts, behavior_type)) as behavior_id,
    user_id,
    item_id,
    category_id,
    CASE behavior_type
        WHEN 'pv'   THEN 'page_view'
        WHEN 'cart' THEN 'add_cart'
        WHEN 'fav'  THEN 'add_fav'
        WHEN 'buy'  THEN 'purchase'
        ELSE 'unknown'
    END as behavior_type,
    from_unixtime(ts) as behavior_time,
    from_unixtime(ts, 'yyyy-MM-dd') as behavior_date,
    CAST(from_unixtime(ts, 'HH') AS INT) as behavior_hour,
    CAST(from_unixtime(ts, 'u') AS INT) as behavior_weekday,
    CASE WHEN CAST(from_unixtime(ts, 'u') AS INT) IN (6,7) THEN 1 ELSE 0 END as is_weekend,
    CASE 
        WHEN user_id IS NULL OR item_id IS NULL OR ts IS NULL THEN 'invalid'
        WHEN behavior_type NOT IN ('pv','cart','fav','buy') THEN 'invalid'
        ELSE 'valid'
    END as data_quality_flag
FROM ods_user_behavior
WHERE dt = '${biz_date}';
```

**DWD层设计要点** ：
- **数据清洗**：空值处理、异常值过滤、枚举值标准化
- **维度退化**：将常用时间维度字段冗余到事实表，减少下游JOIN
- **数据质量标记**：增加`data_quality_flag`字段，便于追溯问题数据
- **行为类型标准化**：将原始编码转换为可读枚举值

#### DWS层 - 数据汇总层

```sql
-- ============================================
-- DWS层：用户行为日汇总表（轻度汇总）
-- ============================================
CREATE TABLE IF NOT EXISTS dws_user_behavior_day (
    user_id                 STRING COMMENT '用户ID',
    dt                      STRING COMMENT '统计日期',
    
    -- 浏览行为指标
    pv_count                BIGINT COMMENT '浏览次数',
    pv_items_distinct       BIGINT COMMENT '浏览商品种类数',
    
    -- 加购行为指标
    cart_count              BIGINT COMMENT '加购次数',
    cart_items_distinct     BIGINT COMMENT '加购商品种类数',
    
    -- 收藏行为指标
    fav_count               BIGINT COMMENT '收藏次数',
    fav_items_distinct      BIGINT COMMENT '收藏商品种类数',
    
    -- 购买行为指标
    purchase_count          BIGINT COMMENT '购买次数',
    purchase_items_distinct BIGINT COMMENT '购买商品种类数',
    purchase_amount         DECIMAL(18,2) COMMENT '购买金额(模拟)',
    
    -- 转化漏斗指标
    cart_rate               DECIMAL(5,4) COMMENT '加购转化率: cart_count/pv_count',
    fav_rate                DECIMAL(5,4) COMMENT '收藏转化率: fav_count/pv_count',
    purchase_rate           DECIMAL(5,4) COMMENT '购买转化率: purchase_count/pv_count',
    
    -- 活跃度指标
    session_count           BIGINT COMMENT '会话数(按30分钟间隔切分)',
    avg_session_duration    INT COMMENT '平均会话时长(秒)'
)
COMMENT '用户行为日汇总表'
PARTITIONED BY (dt STRING)
STORED AS ORC
LOCATION '/warehouse/dws/dws_user_behavior_day';

-- DWS层ETL
INSERT OVERWRITE TABLE dws_user_behavior_day PARTITION(dt='${biz_date}')
SELECT 
    user_id,
    '${biz_date}' as dt,
    SUM(CASE WHEN behavior_type='page_view' THEN 1 ELSE 0 END) as pv_count,
    COUNT(DISTINCT CASE WHEN behavior_type='page_view' THEN item_id END) as pv_items_distinct,
    SUM(CASE WHEN behavior_type='add_cart' THEN 1 ELSE 0 END) as cart_count,
    COUNT(DISTINCT CASE WHEN behavior_type='add_cart' THEN item_id END) as cart_items_distinct,
    SUM(CASE WHEN behavior_type='add_fav' THEN 1 ELSE 0 END) as fav_count,
    COUNT(DISTINCT CASE WHEN behavior_type='add_fav' THEN item_id END) as fav_items_distinct,
    SUM(CASE WHEN behavior_type='purchase' THEN 1 ELSE 0 END) as purchase_count,
    COUNT(DISTINCT CASE WHEN behavior_type='purchase' THEN item_id END) as purchase_items_distinct,
    SUM(CASE WHEN behavior_type='purchase' THEN item_price ELSE 0 END) as purchase_amount,
    -- 转化率计算（避免除零）
    ROUND(SUM(CASE WHEN behavior_type='add_cart' THEN 1 ELSE 0 END) 
          / NULLIF(SUM(CASE WHEN behavior_type='page_view' THEN 1 ELSE 0 END), 0), 4) as cart_rate,
    ROUND(SUM(CASE WHEN behavior_type='add_fav' THEN 1 ELSE 0 END) 
          / NULLIF(SUM(CASE WHEN behavior_type='page_view' THEN 1 ELSE 0 END), 0), 4) as fav_rate,
    ROUND(SUM(CASE WHEN behavior_type='purchase' THEN 1 ELSE 0 END) 
          / NULLIF(SUM(CASE WHEN behavior_type='page_view' THEN 1 ELSE 0 END), 0), 4) as purchase_rate,
    -- 会话数（简化版：按30分钟无操作切分）
    COUNT(DISTINCT session_id) as session_count,
    AVG(session_duration) as avg_session_duration
FROM (
    SELECT *,
           -- 会话切分逻辑
           SUM(CASE WHEN behavior_time - LAG(behavior_time) OVER 
               (PARTITION BY user_id ORDER BY behavior_time) > 1800 THEN 1 ELSE 0 END) 
               OVER (PARTITION BY user_id ORDER BY behavior_time) as session_id
    FROM dwd_user_behavior_fact
    WHERE dt = '${biz_date}' AND data_quality_flag = 'valid'
) t
GROUP BY user_id;

-- ============================================
-- DWS层：商品日汇总表
-- ============================================
CREATE TABLE IF NOT EXISTS dws_item_behavior_day (
    item_id                 STRING COMMENT '商品ID',
    category_id             STRING COMMENT '类目ID',
    dt                      STRING COMMENT '统计日期',
    
    pv_count                BIGINT COMMENT '浏览次数',
    pv_users_distinct       BIGINT COMMENT '浏览用户数',
    cart_count              BIGINT COMMENT '加购次数',
    fav_count               BIGINT COMMENT '收藏次数',
    purchase_count          BIGINT COMMENT '购买次数',
    purchase_users_distinct BIGINT COMMENT '购买用户数',
    
    -- 热度综合评分 (加权计算)
    heat_score              DECIMAL(10,2) COMMENT '商品热度分: 0.4*pv + 0.2*cart + 0.2*fav + 0.2*purchase'
)
COMMENT '商品行为日汇总表'
PARTITIONED BY (dt STRING)
STORED AS ORC
LOCATION '/warehouse/dws/dws_item_behavior_day';
```

#### ADS层 - 应用数据层

```sql
-- ============================================
-- ADS层：RFM用户价值分层表
-- ============================================
CREATE TABLE IF NOT EXISTS ads_user_rfm_segment (
    user_id         STRING COMMENT '用户ID',
    
    -- RFM原始值
    recency_days    INT COMMENT '最近一次购买距今天数(R)',
    frequency       INT COMMENT '购买频率(F)',
    monetary        DECIMAL(18,2) COMMENT '购买金额(M)',
    
    -- RFM评分 (1-5分，按五分位划分)
    r_score         INT COMMENT 'R评分: 1-5',
    f_score         INT COMMENT 'F评分: 1-5', 
    m_score         INT COMMENT 'M评分: 1-5',
    
    -- RFM综合标签
    rfm_score       STRING COMMENT 'RFM综合分: 如555',
    user_segment    STRING COMMENT '用户分层标签',
    
    -- 分层说明
    segment_desc    STRING COMMENT '分层描述',
    
    -- 统计日期
    dt              STRING COMMENT '统计日期'
)
COMMENT 'RFM用户价值分层表'
PARTITIONED BY (dt STRING)
STORED AS ORC
LOCATION '/warehouse/ads/ads_user_rfm_segment';

-- RFM计算逻辑
INSERT OVERWRITE TABLE ads_user_rfm_segment PARTITION(dt='${stat_date}')
WITH user_rfm_base AS (
    SELECT 
        user_id,
        DATEDIFF('${stat_date}', MAX(behavior_date)) as recency_days,
        COUNT(DISTINCT behavior_date) as frequency,
        COUNT(*) as purchase_count,
        SUM(item_price) as monetary
    FROM dwd_user_behavior_fact
    WHERE behavior_type = 'purchase' 
      AND behavior_date <= '${stat_date}'
      AND behavior_date >= DATE_SUB('${stat_date}', 90)  -- 近90天
    GROUP BY user_id
),
rfm_scored AS (
    SELECT 
        user_id,
        recency_days,
        frequency,
        monetary,
        -- R评分：天数越小分越高（反向）
        NTILE(5) OVER (ORDER BY recency_days ASC) as r_score,
        -- F评分：频率越高分越高
        NTILE(5) OVER (ORDER BY frequency DESC) as f_score,
        -- M评分：金额越高分越高
        NTILE(5) OVER (ORDER BY monetary DESC) as m_score
    FROM user_rfm_base
)
SELECT 
    user_id,
    recency_days,
    frequency,
    monetary,
    r_score,
    f_score,
    m_score,
    CONCAT(r_score, f_score, m_score) as rfm_score,
    CASE 
        WHEN r_score >= 4 AND f_score >= 4 AND m_score >= 4 THEN '重要价值客户'
        WHEN r_score >= 4 AND f_score >= 4 AND m_score < 4 THEN '重要保持客户'
        WHEN r_score >= 4 AND f_score < 4 AND m_score >= 4 THEN '重要发展客户'
        WHEN r_score >= 4 AND f_score < 4 AND m_score < 4 THEN '重要挽留客户'
        WHEN r_score < 4 AND f_score >= 4 AND m_score >= 4 THEN '一般价值客户'
        WHEN r_score < 4 AND f_score >= 4 AND m_score < 4 THEN '一般保持客户'
        WHEN r_score < 4 AND f_score < 4 AND m_score >= 4 THEN '一般发展客户'
        ELSE '流失客户'
    END as user_segment,
    CASE 
        WHEN r_score >= 4 AND f_score >= 4 AND m_score >= 4 THEN '最近购买、高频高消费，核心VIP'
        WHEN r_score >= 4 AND f_score >= 4 THEN '最近购买且高频，需提升客单价'
        WHEN r_score >= 4 AND m_score >= 4 THEN '最近高消费但低频，需提升复购'
        ELSE '详见分层规则'
    END as segment_desc,
    '${stat_date}' as dt
FROM rfm_scored;
```

### 4.2 购物篮关联分析（Apriori算法）

```python
# 购物篮关联分析 - Python实现
from itertools import combinations
from collections import defaultdict, Counter

def apriori_analysis(transactions, min_support=0.01, min_confidence=0.3):
    """
    购物篮关联规则挖掘
    transactions: 交易列表，每个元素是商品ID集合
    """
    # 1. 计算1-项集支持度
    item_counts = Counter()
    for transaction in transactions:
        for item in transaction:
            item_counts[item] += 1
    
    total = len(transactions)
    frequent_1 = {item: count/total for item, count in item_counts.items() 
                  if count/total >= min_support}
    
    # 2. 计算2-项集支持度
    pair_counts = Counter()
    for transaction in transactions:
        items = [item for item in transaction if item in frequent_1]
        for pair in combinations(items, 2):
            pair_counts[tuple(sorted(pair))] += 1
    
    frequent_2 = {pair: count/total for pair, count in pair_counts.items()
                  if count/total >= min_support}
    
    # 3. 生成关联规则
    rules = []
    for (a, b), support in frequent_2.items():
        # A→B 置信度
        conf_a_to_b = pair_counts[(a,b)] / item_counts[a]
        # B→A 置信度
        conf_b_to_a = pair_counts[(a,b)] / item_counts[b]
        
        if conf_a_to_b >= min_confidence:
            lift = conf_a_to_b / frequent_1[b]
            rules.append({
                'antecedent': a,
                'consequent': b,
                'support': support,
                'confidence': conf_a_to_b,
                'lift': lift
            })
    
    return rules

# 应用：分析"购买了商品A的用户，有多大可能购买商品B"
# 结果存入 ads_market_basket_rules 表
```

**关联规则解读**：
- **支持度(Support)**：商品A和B同时出现的订单占比
- **置信度(Confidence)**：买了A的用户中，也买B的比例
- **提升度(Lift)**：买了A后买B的概率 / 单独买B的概率
  - Lift > 1：正相关，可捆绑销售
  - Lift = 1：独立事件
  - Lift < 1：负相关，避免同时推荐

```python
-- RFM用户分层表

    user_id STRING,           -- 用户ID
    r_score INT,              -- R评分1-5
    f_score INT,              -- F评分1-5
    m_score INT,              -- M评分1-5
    rfm_score STRING,         -- RFM综合分
    user_segment STRING,      -- 用户分层
    dt STRING                 -- 统计日期
);

【生成规则】
1. 仅使用上述Schema中存在的表和字段
2. 日期过滤使用 dt 字段，格式为 'yyyy-MM-dd'
3. 聚合函数需配合GROUP BY使用
4. 排序使用ORDER BY，限制条数使用LIMIT
5. 返回纯SQL，不要任何解释

【示例】
Q: 昨天购买次数最多的前5个用户是谁？
A: SELECT user_id, purchase_count FROM dws_user_behavior_day 
   WHERE dt = '2017-12-03' ORDER BY purchase_count DESC LIMIT 5;

Q: {user_question}
A:
"""

def nl2sql(user_question: str, llm_client) -> str:
    """NL2SQL核心函数"""
    prompt = NL2SQL_PROMPT.format(user_question=user_question)
    
    # 调用大模型API（OpenAI/通义千问/DeepSeek等）
    response = llm_client.chat.completions.create(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.1,  # 低温度保证确定性
        max_tokens=500
    )
    
    sql = response.choices[0].message.content.strip()
    
    # SQL安全检查
    sql = sql_safety_check(sql)
    
    return sql

def sql_safety_check(sql: str) -> str:
    """SQL安全检查：防止注入和危险操作"""
    dangerous_keywords = ['DROP', 'DELETE', 'UPDATE', 'INSERT', 'TRUNCATE', 'ALTER']
    for kw in dangerous_keywords:
        if kw in sql.upper():
            raise ValueError(f"检测到危险操作: {kw}")
    return sql
```

```python
   - 四、转化漏斗分析（各环节转化率）
   - 五、行动建议（3条具体可执行建议）
3. 语言风格：专业、简洁、数据驱动
4. 每个结论需附带具体数据支撑
5. 总字数控制在800-1200字

请直接输出报告正文。
"""

def generate_report(data_date: str, llm_client) -> str:
    """生成智能报表"""
    # 1. 从Hive读取数据
    kpis = query_hive(f"SELECT * FROM ads_dashboard_kpi WHERE dt='{data_date}'")
    rfm = query_hive(f"""
        SELECT user_segment, COUNT(*) as cnt 
        FROM ads_user_rfm_segment 
        WHERE dt='{data_date}' 
        GROUP BY user_segment
    """)
    top_items = query_hive(f"""
        SELECT item_id, heat_score 
        FROM dws_item_behavior_day 
        WHERE dt='{data_date}' 
        ORDER BY heat_score DESC LIMIT 10
    """)
    
    # 2. 组装Prompt
    prompt = REPORT_GENERATION_PROMPT.format(
        start_date=data_date,
        end_date=data_date,
        data_date=data_date,
        kpis_json=json.dumps(kpis, ensure_ascii=False),
        rfm_json=json.dumps(rfm, ensure_ascii=False),
        top_items_json=json.dumps(top_items, ensure_ascii=False)
    )
    
    # 3. 调用LLM生成报告
    response = llm_client.chat.completions.create(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.7,
        max_tokens=2000
    )
    
    report_text = response.choices[0].message.content
    
    # 4. 渲染为HTML/PDF
    html_report = render_report_template(report_text, kpis, rfm, top_items)
    
    return html_report
```
