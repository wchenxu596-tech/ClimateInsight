## 4. API 层（src/api/index.js）

### 4.1 Axios实例配置

```javascript
const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',  // 可环境变量覆盖
  timeout: 15000,                                        // 15秒超时
})
api.interceptors.response.use(r => r, err => {
  console.error('[API]', err.response?.data?.message || err.message)
  return Promise.reject(err)                             // 继续抛出给调用方
})
```

### 4.2 7个API函数

| 函数 | 方法 | 路径 | 参数 | 返回值 |
|------|------|------|------|--------|
| `getKPI(year)` | GET | `/kpi` | `{ year }` | `{code, data:[{kpi_name, kpi_value, kpi_unit, kpi_desc}], meta}` |
| `getMonthly(year)` | GET | `/monthly` | `{ year }` | `{code, data:[{obs_month, avg_temp, avg_max, avg_min}], meta}` |
| `getZones(year)` | GET | `/zones` | `{ year }` | `{code, data:[{climate_zone, cnt}], meta}` |
| `getRanking(year, category, limit)` | GET | `/ranking` | `{ year, category, limit }` | `{code, data:[{rank_num, station_id, station_name, value}], meta}` |
| `getTrend(year)` | GET | `/trend` | `{ year }` | 月度趋势（与monthly类似） |
| `getHealth()` | GET | `/health` | — | 健康检查（未在前端调用） |
| `askAgent(question, year)` | POST | `/agent/query` | `{ question, year }` | `{code, data:{answer, intent, table?, chart?, limitations?}, meta}` |

**注意**：`getTrend` 和 `getHealth` 已定义但未在任何页面调用。

### 6.2 导航样式

- 使用原生 `<router-link>` 而非 Element Plus 的 `el-menu`（避免水平菜单自动折叠到"..."的问题）
- 激活态：白色文字 + 蓝色底部边框（`border-bottom-color: #64b5f6`）
- 悬停态：半透明白色背景
- Logo和导航项均设置 `white-space:nowrap; flex-shrink:0` 防止换行

### 6.3 年份选择器

```javascript
const availableYears = [2022, 2023, 2024]
const selectedYear = ref(2024)       // 默认2024年
provide('selectedYear', selectedYear) // 通过依赖注入传递给所有子组件

function onYearChange(year) {
  selectedYear.value = year
}
```

**关键机制**：`<router-view :key="$route.path + selectedYear" />`，年份变化时 `key` 改变，Vue会销毁并重建路由组件，确保数据完全重新加载。

## 8. Dashboard.vue — 全球气候总览

### 8.1 状态管理

| 变量 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `selectedYear` | Ref\<number\> | inject自App | 当前选中年份 |
| `loading` | Ref\<bool\> | true | 加载状态 |
| `error` | Ref\<string\> | '' | 错误信息 |
| `empty` | Ref\<bool\> | false | 空数据状态 |
| `kpis` | Ref\<array\> | 4个默认'--' | KPI卡片数据 |
| `monthlyOption` | Reactive | — | ECharts月度折线图配置 |
| `zoneOption` | Reactive | — | ECharts气候带饼图配置 |
| `hotOption` | Reactive | — | ECharts最热排名柱状图配置 |

### 8.2 加载流程

```
onMounted → load()
watch(selectedYear) → load()   ← 年份切换时重新加载

load():
  1. loading=true, error='', empty=false
  2. Promise.all([getKPI(y), getMonthly(y), getZones(y), getRanking(y,'hottest',15)])
  3. 四个API全部返回后：
     - 如果KPI和monthly都为空 → empty=true
     - 否则分别填充kpis/monthlyOption/zoneOption/hotOption
  4. catch → error='数据加载失败...'
  5. finally → loading=false
```

### 8.3 渲染逻辑

**三个状态分支**（使用 `v-if/v-else-if/v-else` 链）：

1. `loading=true` → `<el-skeleton :rows="8" animated />` 骨架屏
2. `error!=''` → `<el-alert type="error" show-icon closable>` + 重试按钮
3. `empty=true` → `<el-empty description="该年份暂无数据" />`
4. 正常数据 → KPI卡片 + 三个图表

### 8.4 KPI卡片

```html
<div class="kpi-row">   ← flex布局，自动换行
  <div class="kpi-card" v-for="k in kpis">
    <div class="kpi-value">{{ k.value }}</div>    ← 大号数字 32px
    <div class="kpi-label">{{ k.label }}</div>    ← 灰色标签 13px
  </div>
</div>
```

KPI映射关系（从 `ads_kpi` 表）：
| kpi_name | 显示标签 | 显示值 |
|----------|----------|--------|
| global_avg_temp | 全球年均温 | 13.97°C |
| total_stations | 活跃气象站 | 12154 |
| extreme_event_pct | 极端天气占比 | 16.05% |
| hottest_station_temp | 年度最高温 | 56.0°C |

```javascript

// ...
```

## 12. BIAgent.vue — AI分析助手（全局悬浮组件）

### 12.1 两种状态

**折叠态**：右下角固定悬浮圆形按钮
```html
<div class="agent-fab" @click="isOpen=true">
  <span>🤖</span>
</div>
```
- 52×52px圆形，深蓝渐变背景，阴影
- hover时放大1.08倍
- z-index: 9999

**展开态**：可拖拽聊天面板
```html
<div class="agent-panel" :style="{left:pos.x, top:pos.y, width:size.w, height:size.h}">
  <!-- 标题栏（拖拽手柄） -->
  <div class="panel-header" @mousedown="onDragStart">
    <span>🤖 AI 分析助手</span>
    <el-button @click.stop="isOpen=false">−</el-button>   ← 关闭
  </div>
  <!-- 消息列表 -->
  <div class="panel-messages" ref="msgBox">...</div>
  <!-- 输入栏 -->
  <div class="panel-input">
    <el-input v-model="question" @keyup.enter="send" maxlength="300"/>
    <el-button @click="send">发送</el-button>
  </div>
  <!-- 快捷问题标签 -->
  <div class="panel-quick">
    <el-tag v-for="q in quickQuestions" @click="question=q;send()">{{ q }}</el-tag>
  </div>
  <!-- 缩放拖拽手柄（右下角） -->
  <div class="resize-handle" @mousedown.stop="onResizeStart"></div>
</div>
```

### 12.2 关键状态

| 变量 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `isOpen` | Ref\<bool\> | false | 面板是否展开 |
| `question` | Ref\<string\> | '' | 输入框内容 |
| `loading` | Ref\<bool\> | false | 等待后端回复 |
| `messages` | Ref\<array\> | [{role:'assistant', content:'👋 你好！...'}] | 消息历史 |
| `pos` | Reactive{x,y} | 右下角 | 面板位置 |
| `size` | Reactive{w,h} | {380, 460} | 面板尺寸 |
| `selectedYear` | inject | 2024 | 当前全局年份 |

### 12.3 拖拽实现

```
mousedown(onDragStart) → 记录起始位置
mousemove(onDragMove)  → 更新pos.x, pos.y（限制在视口内）
mouseup(onDragEnd)     → 移除事件监听
```

### 12.4 缩放实现

```
mousedown(onResizeStart) → 记录起始尺寸
mousemove(onResizeMove)  → 更新size.w, size.h
  w: Math.max(300, Math.min(rezStartW + deltaX, 700))   // 300~700px
  h: Math.max(260, Math.min(rezStartH + deltaY, 700))   // 260~700px
mouseup(onResizeEnd)     → 移除事件监听
```

右下角16×16px的斜纹手柄，cursor: nwse-resize。

### 12.5 滚轮隔离

面板的 `@wheel.stop` 阻止事件冒泡到外层页面。消息区到边界时 `preventDefault` 防止穿透。

### 12.6 发送消息

```
send():
  1. 校验：question非空 && !loading
  2. messages.push({role:'user', content:q})
  3. question='', loading=true, scrollBottom()
  4. await askAgent(q, selectedYear.value)        ← POST /api/agent/query
  5. 解析响应：
     - d.answer → msg.content
     - d.table → msg.table (rows转为对象数组，交给el-table)
     - d.chart → msg.chart (调用chartToOption转为ECharts配置)
     - d.limitations → 追加 ⚠️ 提示到content末尾
  6. messages.push(msg)
  7. catch → '分析助手暂不可用...'
  8. loading=false, scrollBottom()
```

### 12.7 图表适配（chartToOption）

后端返回：`{type, x, y, name}` → 转为ECharts option：

```javascript
bar  → { tooltip, xAxis:{data:x}, yAxis:{name}, series:[{type:'bar', data:y}] }
line → { tooltip, xAxis:{data:x}, yAxis:{name}, series:[{type:'line', data:y, smooth:true}] }
pie  → { tooltip, series:[{type:'pie', radius:'55%', data:x.map((n,i)=>({name:n, value:y[i]}))}] }
```

tooltip全部使用 `valueFormatter: v=>v.toFixed(2)`。

### 12.8 快捷问题


## 14. 错误处理模式

所有页面统一使用以下模式：

```javascript
const loading = ref(true)     // 初始加载中
const error = ref('')         // 错误信息
const empty/hasData = ref(false)  // 空数据标记

async function load() {
  loading.value = true; error.value = ''; empty.value = false
  try {
    const res = await someAPI(...)
    if (!res.data?.data?.length) { empty.value = true; return }
    // 填充数据...
  } catch(e) {
    error.value = '数据加载失败，请确认后端已启动 (localhost:5000)'
    console.error(e)
  } finally {
    loading.value = false
  }
}

watch(selectedYear, load)    // 年份变化 → 重新加载
onMounted(load)               // 首次挂载 → 加载
```

**注意**：`router-view` 的 `:key` 绑定确保了年份切换时组件重建，但如果组件本身有 `watch(selectedYear)` 也会再次触发加载。这是双重保险。
