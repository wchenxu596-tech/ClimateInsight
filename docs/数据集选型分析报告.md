## 3. 方案一：全球气候变化数据 🌍

### 3.1 基本信息

| 属性                 | 详情                                                                   |
| -------------------- | ---------------------------------------------------------------------- |
| **数据集全称** | NOAA Global Historical Climatology Network Daily (GHCN-Daily)          |
| **维护机构**   | 美国国家海洋和大气管理局 (NOAA) / 国家环境信息中心 (NCEI)              |
| **数据规模**   | 10万+气象站 × 365天 × 多年 =**数亿行**（选10年约3亿行）        |
| **时间跨度**   | 1880年 — 至今（**140年+**）                                     |
| **更新频率**   | 每日更新                                                               |
| **字段结构**   | 气象站ID、日期、观测类型(TMAX/TMIN/PRCP/SNOW...)、观测值、经纬度、海拔 |

### 3.2 数据字段详解

| 字段             | 类型    | 说明                         | 示例                                                               |
| ---------------- | ------- | ---------------------------- | ------------------------------------------------------------------ |
| `station_id`   | STRING  | 气象站唯一标识               | USW00094728                                                        |
| `date`         | DATE    | 观测日期                     | 2023-07-15                                                         |
| `element`      | STRING  | 观测要素                     | TMAX(最高温)/TMIN(最低温)/PRCP(降水量)/SNOW(降雪量)/SNWD(积雪深度) |
| `value`        | INTEGER | 观测值（需除以10恢复真实值） | 325 → 32.5°C                                                     |
| `latitude`     | FLOAT   | 纬度                         | 39.9042                                                            |
| `longitude`    | FLOAT   | 经度                         | 116.4074                                                           |
| `elevation`    | FLOAT   | 海拔(米)                     | 54.0                                                               |
| `station_name` | STRING  | 气象站名称                   | BEIJING                                                            |

### 3.3 下载方式

**方式一：NOAA 官方直接下载（推荐）**

```
# 按年份下载 CSV 文件
# 格式：https://www.ncei.noaa.gov/data/global-summary-of-the-day/archive/{year}.tar.gz

# 示例（2023年全年数据）：
https://www.ncei.noaa.gov/data/global-summary-of-the-day/archive/2023.tar.gz

# 解压后为 365 个 CSV 文件，每个文件包含当天全球所有气象站数据
# 单日文件约 10-30MB，全年约 3-8GB（取决于气象站密度）
```

**方式二：AWS Open Data（更快）**

```
# AWS 公开数据集，无需 AWS 账号即可下载
aws s3 sync s3://noaa-gsod-pds/2023/ ./gsod_2023/ --no-sign-request

# 或直接用 HTTPS（每个站点一个CSV）：
# 数据在 S3 上以 Parquet 格式提供
```

**方式三：Kaggle 镜像（最简单）**

```
# Kaggle 上有整理好的版本：
https://www.kaggle.com/datasets/noaa/noaa-global-historical-climatology-network-daily

# 需要 Kaggle 账号（免费），下载为 CSV/ZIP
# 也可通过 Kaggle API 下载：
kaggle datasets download noaa/noaa-global-historical-climatology-network-daily
```

> **✅ 验证状态**：NOAA 官方 GSOD archive 链接模式已验证可用（截至2026年7月）

### 3.4 适合做的分析

| 分析类型               | 技术实现                                             | 说明                                |
| ---------------------- | ---------------------------------------------------- | ----------------------------------- |
| **全球变暖趋势** | ADS层按年聚合全球平均温度                            | 折线图展示140年升温曲线，极具冲击力 |
| **极端天气事件** | ADS层筛选温度>40°C或降水量>100mm的天数              | 柱状图展示极端事件频率变化          |
| **城市对比**     | 选定10个代表性城市，按年聚合对比                     | "北京vs纽约vs伦敦vs新德里"气温对比  |
| **季节模式**     | 按月份聚合多年平均                                   | 热力图展示各月平均温度              |
| **RFM改编**      | R=最近一次极端事件日期, F=极端事件频次, M=累积降水量 | 城市气候风险分层                    |

### 3.5 项目叙事角度

> **"用数据讲述气候变化"** — 140年的温度数据，见证了工业革命以来地球的完整变暖历程。这个项目不只是技术展示，更是一次基于数据的科学传播。

**答辩开场白示例**：

> "从1880年到2024年，全球平均气温上升了约1.2°C。这1.2°C背后，是10万个气象站、140年不间断的观测记录。我们的平台将这些数据转化为每个人都能看懂的洞察，帮助你理解气候变化到底如何影响你所在的城市。"

### 3.6 AI交互示例

```
用户：过去50年全球平均气温上升了多少？
系统：自动生成SQL → AVG聚合 → 折线图 + "自1975年以来上升约0.8°C"

用户：2023年高温天数最多的10个中国城市？
系统：SELECT city, COUNT(*) WHERE TMAX>35 GROUP BY city → 柱状图

用户：北京的冬季平均温度变化趋势？
系统：WHERE station='BEIJING' AND month IN (12,1,2) GROUP BY year → 趋势线

用户：全球哪些地区降水减少最严重？
系统：对比前后30年平均降水量差异 → 散点地图
```

### 3.7 优缺点

| ✅ 优点                                            | ❌ 缺点                                        |
| -------------------------------------------------- | ---------------------------------------------- |
| 数据量巨大（选10年=3亿行），充分验证大数据处理能力 | 原始数据需要预处理（除以10、过滤异常值）       |
| 140年时间跨度，趋势分析极具说服力                  | 全球气象站数据量不均匀（欧美密集、非洲稀疏）   |
| 气候变化是全球热点话题，答辩时容易引发共鸣         | 温度/降水不如电商的RFM直观，需要"换算"分析模型 |
| 支持地理可视化（地图热力图），比电商多了空间维度   | 缺少"用户"概念，需要把气象站类比为用户         |
| NOAA 官方数据权威性无可质疑                        | —                                             |

## 4. 方案二：全球能源转型数据 ⚡

### 4.1 基本信息

| 属性                 | 详情                                                                                                   |
| -------------------- | ------------------------------------------------------------------------------------------------------ |
| **数据集全称** | Our World in Data — Energy Dataset                                                                    |
| **维护机构**   | Our World in Data (牛津大学附属) / 数据来源包括BP、EIA、IEA                                            |
| **数据规模**   | 200+国家 × 60年 × 129个指标 =**约2.5万行**（原始宽表） → 经Long-form转换后**约150万行** |
| **时间跨度**   | 1965年 — 2024年（60年）                                                                               |
| **更新频率**   | 每年更新                                                                                               |
| **文件大小**   | CSV 约 5MB（原始宽表格式）                                                                             |

### 4.2 核心字段

| 字段类别 | 字段名                                                             | 说明                    |
| -------- | ------------------------------------------------------------------ | ----------------------- |
| 基础     | `country`, `year`, `iso_code`                                | 国家、年份、ISO代码     |
| 化石能源 | `coal_production`, `oil_production`, `gas_production`        | 煤/石油/天然气产量(TWh) |
| 可再生   | `solar_consumption`, `wind_consumption`, `hydro_consumption` | 太阳能/风能/水力        |
| 碳排放   | `co2`, `co2_per_capita`, `cumulative_co2`                    | 碳排放总量/人均/累计    |
| 电力     | `electricity_generation`, `electricity_share_renewables`       | 发电量/可再生占比       |
| 人口经济 | `population`, `gdp`                                            | 人口/GDP                |

### 4.3 下载方式

**方式一：GitHub 直接下载（最简单，✅ 已验证）**

```
# CSV 文件（直接可用）：
https://owid-public.owid.io/data/energy/owid-energy-data.csv

# XLSX 文件：
https://owid-public.owid.io/data/energy/owid-energy-data.xlsx

# 数据字典 (Codebook)：
https://github.com/owid/energy-data/blob/master/owid-energy-codebook.csv

# GitHub 仓库：
https://github.com/owid/energy-data
```

**方式二：Our World in Data 网站**

```
https://ourworldindata.org/energy
# 页面上的图表数据均可下载为 CSV
```

> **✅ 验证状态**：owid-energy-data.csv 链接已验证，可直接下载

### 4.4 适合做的分析

| 分析类型                     | 技术实现                               | 说明                     |
| ---------------------------- | -------------------------------------- | ------------------------ |
| **可再生能源占比趋势** | 按国家+年份聚合 renewable_share        | 折线图展示各国转型进度   |
| **碳排放达峰分析**     | 检测每个国家 CO2 峰值的年份            | "已有49个国家实现碳达峰" |
| **人均对比**           | 人均能耗、人均排放排名                 | 横向柱状图               |
| **能源结构变迁**       | 各国煤/油/气/可再生占比的 Stacked Area | 展示 60 年能源结构变化   |
| **GDP vs 碳排放解耦**  | 散点图展示经济增长与碳排放脱钩的国家   | 可持续发展分析           |

### 4.5 项目叙事角度

> **"从煤到光：全球能源转型的数字化见证"** — 60年数据记录了人类能源利用方式的根本变革。哪些国家走在前列？哪些还在依赖化石燃料？

### 4.6 AI交互示例

```
用户：2023年可再生能源发电占比最高的10个国家？
用户：中国过去20年太阳能发电增长了多少倍？
用户：哪些国家已经实现了碳达峰？
用户：人均碳排放最高的国家和最低的国家差多少倍？
```

### 4.7 优缺点

| ✅ 优点                            | ❌ 缺点                                                               |
| ---------------------------------- | --------------------------------------------------------------------- |
| CSV直接可下，无需预处理，5MB秒下载 | 原始数据是宽表格式（字段很多），需要转为Long-form才能在Hive中高效存储 |
| 数据质量极高，OWID团队专业清洗过   | 绝对行数偏少（万级），虽然转长表后可达百万级                          |
| 60年时间跨度，趋势分析有力         | 缺少"用户"粒度，分析以国家为单位                                      |
| 碳排放、能源转型是顶级热门话题     | 部分国家数据不完整（早期年份缺失）                                    |
| GitHub 托管，下载稳定可靠          | —                                                                    |

## 5. 方案三：美国纽约出租车数据 🚕

### 5.1 基本信息

| 属性                 | 详情                                        |
| -------------------- | ------------------------------------------- |
| **数据集全称** | NYC TLC Trip Record Data                    |
| **维护机构**   | 纽约市出租车与礼车委员会 (NYC TLC)          |
| **数据规模**   | 每月约1000万行，选12个月 =**1.2亿行** |
| **时间跨度**   | 2009年 — 至今（17年）                      |
| **更新频率**   | 每月更新                                    |
| **格式**       | Parquet（列式压缩，Hive原生支持）           |
| **文件大小**   | 每月约 200-300MB，12个月 ≈ 3GB             |

### 5.2 核心字段

| 字段                      | 类型      | 说明                          |
| ------------------------- | --------- | ----------------------------- |
| `tpep_pickup_datetime`  | TIMESTAMP | 上车时间                      |
| `tpep_dropoff_datetime` | TIMESTAMP | 下车时间                      |
| `passenger_count`       | INT       | 乘客人数                      |
| `trip_distance`         | FLOAT     | 行程距离（英里）              |
| `PULocationID`          | INT       | 上车地点区域ID                |
| `DOLocationID`          | INT       | 下车地点区域ID                |
| `fare_amount`           | DECIMAL   | 基本车费                      |
| `tip_amount`            | DECIMAL   | 小费                          |
| `total_amount`          | DECIMAL   | 总金额                        |
| `payment_type`          | INT       | 支付方式(1=信用卡/2=现金/...) |
| `rate_code_id`          | INT       | 计价类型                      |

### 5.3 下载方式

**✅ 已验证：CloudFront 直接下载**

```
# 黄色出租车，按月下载 Parquet 文件
# URL 格式：https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_{YYYY}-{MM}.parquet

# 示例 — 2023年全年12个文件：
https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_2023-01.parquet
https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_2023-02.parquet
...
https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_2023-12.parquet

# 批量下载（PowerShell）：
1..12 | ForEach-Object { 
    $m = $_.ToString('00')
    Invoke-WebRequest "https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_2023-$m.parquet" -OutFile "yellow_2023-$m.parquet"
}

# 数据字典：
https://www.nyc.gov/assets/tlc/downloads/pdf/data_dictionary_trip_records_yellow.pdf

# 区域对照表（LocationID ↔ 区域名）：
https://d37ci6vzurychx.cloudfront.net/misc/taxi_zone_lookup.csv
```

> **✅ 验证状态**：URL 格式已验证可用，CloudFront 全球 CDN 访问速度快

### 5.4 优缺点

| ✅ 优点                      | ❌ 缺点                               |
| ---------------------------- | ------------------------------------- |
| 数据量巨大，Parquet 格式高效 | 社会意义较弱——仅是纽约一城的出租车  |
| 字段丰富，天然适合做RFM      | 答辩时故事性不强，不如气候/能源有高度 |
| 下载速度快，格式立即可用     | 仅覆盖一个城市，缺少跨地域对比        |
| 官方长期维护，稳定性好       | —                                    |

## 7. 方案五：全球粮食价格数据 🌾

### 7.1 基本信息

| 属性                 | 详情                                                     |
| -------------------- | -------------------------------------------------------- |
| **数据集全称** | WFP Global Food Prices Database                          |
| **维护机构**   | 世界粮食计划署 (WFP)                                     |
| **数据规模**   | 数千市场 × 数十商品 × 月度 × 20年 =**数百万行** |
| **时间跨度**   | 2000年 — 至今（20年+）                                  |
| **更新频率**   | 季度更新                                                 |

### 7.2 核心字段

| 字段                       | 类型    | 说明                      |
| -------------------------- | ------- | ------------------------- |
| `country`                | STRING  | 国家                      |
| `market`                 | STRING  | 市场名称                  |
| `commodity`              | STRING  | 商品（大米/小麦/玉米/...) |
| `price`                  | DECIMAL | 价格（当地货币）          |
| `currency`               | STRING  | 货币单位                  |
| `unit`                   | STRING  | 计量单位(KG/LB等)         |
| `date`                   | DATE    | 日期（月度）              |
| `latitude`/`longitude` | FLOAT   | 市场经纬度                |

### 7.3 下载方式

```
# 方式一：HDX 平台搜索
https://data.humdata.org/dataset/?q=wfp+food+prices

# 方式二：Kaggle 镜像
https://www.kaggle.com/datasets/jboysen/global-food-prices

# 方式三：WFP 官方 API
https://data.vam.wfp.org/
```

## 10. 最终推荐

### 🥇 首选：全球气候变化数据（NOAA GHCN）

> **一句话理由**：140年的数据规模、气候变化的社会意义、10万+气象站的丰富维度——这是最能同时满足"技术难度"和"社会价值"的选择。

**建议选10年数据（2015-2024）**：

- 约3亿行记录，充分展示大数据处理能力
- 10年跨度可做趋势分析和季节模式
- 时间近，贴近当下气候议题
- 文件总大小约3-5GB，与现有CSV相当

### 🥈 备选：全球能源转型数据（Our World in Data）

> 如果担心气候数据预处理太复杂，能源数据是"开箱即用"的最优解——CSV直下5MB，数据质量极高，60年跨度足够震撼。

**建议**：将宽表转为Long-form（国家×年份×指标类型×值），Hive中按指标类别分区，可轻松扩展到百万行。

### 🥉 保底：纽约出租车数据

> 数据量撑得起场面，格式不折腾，但社会意义偏弱——适合作为"如果前两个都不行"的保底方案。
