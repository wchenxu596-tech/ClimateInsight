# 实施方案 v3.0（基于NOAA气候数据）

---

**项目名称**：ClimateInsight — 全球气候智能分析平台  
**Slogan**：用140年数据讲述气候变化  
**数据集**：NOAA GSOD — 2024年单年 = **1.04亿行 × 12,160气象站**  
**周期**：7.21—7.25（5天）

## 三、数据仓库设计

### 3.1 四层结构

```
ODS: ods_climate_daily        (原始数据, 按year分区, ~19字段)
  │  每年1.04亿行, TEXTFILE+CSV
  ▼
DWD: dwd_climate_daily        (清洗+单位转换+衍生字段, ~35字段)
  │  温度°C, 降水mm, 极端标记, 季节, 气象标记
  ▼
DWS: dws_station_monthly      (气象站×月聚合, ~25字段)
  │  月均温度, 月降水, 极端天数, 气候带分类
  ▼
ADS: 3张应用表
  ├── ads_global_temp_trend   (全球年均温趋势)
  ├── ads_city_climate_ranking (最热/最冷/最多雨排名)
  └── ads_climate_kpi          (大屏KPI快照)
```

### 3.2 关键数据分析场景

| 场景 | SQL | 对应图表 |
|------|-----|----------|
| 全球年均温趋势 | `SELECT year, AVG(avg_temp) FROM dws_station_monthly GROUP BY year` | 折线图 |
| 最热/最冷城市 | `ORDER BY avg_temp DESC/ASC LIMIT 10` | 横向柱状图 |
| 极端天气日数趋势 | `SELECT year, SUM(extreme_days) FROM dws_station_monthly` | 面积图 |
| 城市气候对比 | 选取5-10个代表性城市 | 折线图对比 |
| 季节模式 | 按月聚合多年均值 | 热力图 |
| 气候带分布 | `GROUP BY climate_zone` | 饼图 |

### 3.3 "RFM"在气候中的等价

| 原电商RFM | 气候等价 | 含义 |
|-----------|----------|------|
| R (Recency) | 最近一次极端天气距今天数 | 该站是否"活跃"于极端事件 |
| F (Frequency) | 极端天气事件频次 | 该站多容易受极端天气影响 |
| M (Monetary) → I (Intensity) | 极端天气强度（最高温/最大降水） | 该站极端天气的严重程度 |

→ 产出：**气象站气候风险分层**（高风险/中风险/低风险/安全）

## 五、AI交互设计

### NL2SQL 典型问答

```
🌡️ 温度类：
  "过去10年全球平均气温上升了多少？"
  "2024年最热的10个城市有哪些？"
  "北京过去5年夏季平均气温变化？"

🌧️ 降水类：
  "全球年降水量最高的气象站在哪里？"
  "哪些国家暴雨天数近年在增加？"
  "2024年降水量超过2000mm的城市？"

⚠️ 极端天气类：
  "极端天气事件最多的10个气象站？"
  "过去20年极端高温天数变化趋势？"
  "雷暴天数最多的地区是哪些？"

🌍 气候对比类：
  "热带和温带地区平均温差多少？"
  "南半球和北半球气温变化趋势对比？"
  "海拔3000米以上气象站的平均温度？"
```

### 智能报告模板

```

        创下有记录以来最热年份"

   [折线图: 10年温度变化]
```

## 六、和原电商方案的对照

| 维度 | 原(电商) | 新(气候) | 变化 |
|------|----------|----------|------|
| 数据集 | 淘宝1亿行×9天 | NOAA 1亿行×每年 | ✅ 时间跨度质的飞跃 |
| 金额字段 | 需模拟伪造 | **不需要** — 温度/降水是真实数值 | ✅ 消除数据造假顾虑 |
| RFM → RFI | R=最近购买 F=频率 M=金额 | R=极端事件间隔 F=极端频次 I=极端强度 | ✅ 同样三层打分逻辑 |
| 关联规则 | 类目关联→弃用 | 气候带特征关联分析 | ✅ 更自然的业务含义 |
| 前端名称 | 电商数据大屏 | **ClimateInsight 气候智能平台** | ✅ 更有格调 |
| 答辩故事 | 电商分析(普通) | **气候变化(高格局)** | ✅ 质的飞跃 |
